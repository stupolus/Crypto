# Retro 2026-05-18 — composite: гейт-валидация ЗАБЛОКИРОВАНА тарифом Coinglass

План: 31-34. Ключ Coinglass получен → попытка пройти гейт
(backfill → backtest → критерий iter#4).

## Что выяснилось про тариф Coinglass

Ключ валиден, но план **сильно ограничен**:

| Ограничение | Факт |
|---|---|
| Интервалы | 15m/1h → **403** (не на плане). 4h/12h/1d → ок |
| Глубина истории | liq/OI/CVD — только с **~2025-11-19** (400 «earliest allowed start_time»). ~6 мес максимум |
| Rate limit | агрессивный **429** уже на 2-м прогоне подряд |

## Что показали прогоны (честно — невалидно)

- **BTC 4h** (единственный частично прошедший): IS 2592 свечей → **0 сделок** (ранний период без Coinglass-данных), OOS 1728 → **2 сделки**. PF 0.0 (вырожденно, нет убыточных на выборке из 2), «+4.2%, 100% winrate» — **статистический мусор**, тот же урок малого sample, что мы отвергали всю сессию.
- **ETH 4h**: 429 на всех эндпоинтах → пусто → honest-gate скрипта корректно вышел без фейковых чисел.

## Вердикт

**Composite НЕ провалил и НЕ прошёл критерий — его невозможно честно проверить на этом тарифе Coinglass.** Причина — не стратегия, а данные:
1. Нет интрадей (<4h) → нельзя проверить на родном 15m-таймфрейме.
2. История ~6 мес, ранняя часть пустая → IS без сделок, OOS 2 сделки.
3. 429 рвут backfill на повторных прогонах.

2 сделки = ноль информации (дисциплина iter#3/D3: малый sample = шум). Натягивать «+4.2%» как успех — запрещено и было бы ложью.

## Что сделано правильно

- Honest-gate `composite_backtest` сработал: при пустом ответе/429 — **выход без фейковых чисел**, не выдал мусор за результат.
- Ключ использован только как env-переменная: **не закоммичен, не в файлах, не в PR/логах**.

## Развилка (за пользователем)

- **A. Апгрейд тарифа Coinglass** (интрадей 15m + история ≥12 мес + выше rate-limit) → turnkey `composite_backtest` сразу даёт честный вердикт по iter#4. Это единственный путь к валидной проверке composite.
- **B. Не валидируем/не деплоим composite** на текущем тарифе — по дисциплине (нет валидации → нет демо). Composite остаётся построенной, но «на полке».
