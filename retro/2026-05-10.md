## Часть 2. Развёртывание VPS-инфраструктуры

### Что было
- Работа велась только на Mac, в worktree `dazzling-heyrovsky-629ddf`. Сессия Claude Code локальная — недоступна с iPhone/iPad/другого Mac.
- Пользователь хочет: репо в GitHub со всей историей, доступ с любого устройства, общий VPS для разработки и (позже) деплоя бота.

### Что стало
- **VPS на Hostinger:** `187.124.41.13`, Ubuntu 25.10 (Questing Quokka), 31 GB RAM, 387 GB диск, x86_64. Уже занят боевыми сервисами Albert (coach-bot, cliproxy, mtg, ruflo, nginx, docker, prometheus, grafana, postgres, odoo, outline) — мы их **не трогаем ни при каком обстоятельстве**.
- **SSH-доступ:**
  - Mac → VPS: ключ `~/.ssh/id_ed25519` (ed25519, без passphrase). Добавлен на VPS через hPanel.
  - VPS → GitHub: отдельный ключ `/root/.ssh/github_crypto_ed25519` (никогда не путать с ключом для входа на VPS). Прописан в `/root/.ssh/config` для `Host github.com`. Подтверждено: `ssh -T git@github.com` → `Hi stupolus!`.
- **Репо** склонирован в `/root/Crypto` на VPS, ветка `main`.
- **tmux-workflow:** `ops/crypto-shell.sh` создаёт/подключает к постоянной сессии `crypto` в `/root/Crypto`. С любого устройства: `ssh root@187.124.41.13 -t /root/Crypto/ops/crypto-shell.sh`.
- **gh CLI** локально на Mac залогинен (`gho_...`, scopes `repo + admin:public_key`). Следующие PR-ы Claude может делать без участия пользователя.

### Какую проблему решил
Cross-device доступ к работе с Claude Code: смысл этой сессии теперь лежит в репо (этот файл), а будущие Claude-сессии можно запускать на VPS в tmux и подключаться к ним с Mac, iMac, iPhone — это будет одна и та же живая сессия, не пропадающая при отключении.

## Зафиксированные риски (не блокеры сейчас, но к запуску live нужно решить)

1. **Ubuntu 25.10 non-LTS.** Поддержка истекает в июле 2026. Для прод-бота 24/7 — рискованно. **Решение до live-торговли:** мигрировать VPS на Ubuntu 24.04 LTS (поддержка до 2029). Hostinger позволяет переустановку. Записать в `plans/00` как блокер фазы live.
2. **Shared VPS с боевыми сервисами Albert.** Авария в `coach-bot`, `odoo`, `postgres` или OOM в `grafana` положит и трейдинг-бота. **Решение до live:** отдельный VPS строго под трейдинг, желательно в азиатском дата-центре (близость к биржам).
3. **SSH-ключ Mac без passphrase.** Удобно, но компрометация Mac = доступ к VPS. **Решение:** перед запуском live — `ssh-keygen -p -f ~/.ssh/id_ed25519` с passphrase + Mac Keychain.
4. **Пароль root VPS был отправлен в чат-сессию.** Пользователь решил не менять, взял ответственность. Перед live этот пароль обязательно сменить + отключить парольный SSH-логин (`PasswordAuthentication no` в `/etc/ssh/sshd_config`).
5. **Бэкап репо только в GitHub.** Если аккаунт GitHub потеряется — потеряется всё. **Решение позже:** второй remote (GitLab/self-hosted), периодический `git push --all` в оба.

## Решения, принятые на будущее

- **Оркестрация процессов** — `systemd`-юниты (как уже сделано для `coach-bot.service` на этом же VPS). Не Docker, не PM2, не Airflow. Применять начиная с фазы 1 (demo на VST), когда бот должен работать 24/7. До этого момента — запуск руками.
- **Merge PR в main** — теперь делает Claude через `gh pr merge` (gh залогинен на Mac). Раньше это была зона пользователя — упрощаем, потому что пользователь не разбирается в UI GitHub. CLAUDE.md обновить отдельным коммитом, если решим закрепить это правило.

- Не настроен `claude` под отдельным юзером `crypto` на VPS — работаем под `root` (вариант 1). Принято осознанно.
- Не настроен fail2ban / firewall — но они актуальны для боевых сервисов Albert, не для текущей фазы разработки. Перед live — отдельный sweep.
- Не убрана возможность входа на VPS по паролю — это сделаем перед live.
- Не выложен SSH-клиент на iPhone — пользователь сам поставит Blink/Termius, когда будет нужно.
