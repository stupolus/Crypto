завёл синтетические S&P500/акции с короткой историей; правильная рамка
— искать edge на реальном инструменте по длинной внешней истории,
BingX = только исполнение.

## Findings: btc_breakout НЕ переносится на daily equities

Прогон btc_breakout (equity-вариант, volume-фильтр отключён a-priori
как структурно неприменимый к индексу) выявил 3 структурных
несовместимости — это не «нет edge у актива», это «не та стратегия»:

1. **Volume-фильтр.** `volume_multiplier=1.5` калиброван под спайки 15m
   крипты. На дневном GSPC проходит лишь **97 из 2126** пробоев (~4.5%)
   → почти ноль сделок. Отключён в equity-конфиге (pre-registered).
2. **Negative-TP на penny-эпохе.** Сплит-несведённые цены AAPL 1980-х
   (~$0.1): для SHORT `tp1 = entry − r·distance` уходит < 0 →
   `OrderRequest` валидатор падает (`attached_take_profit must be > 0`).
   Стратегия неявно предполагает крипто-диапазон цен.
3. **Warmup vs OOS-окно.** btc_breakout требует ~201 бар прогрева
   (`atr_percentile_lookback=200`). OOS-окно WF = 90 дневных баров < 201
   → в OOS **0 сделок by construction** (IS=365 > 201 → сделки есть).

Посчитанные IS-метрики GSPC (224 окна): IS PF mean **0.52**,
IS PnL mean −0.12% → даже там, где сделки были, edge отрицательный.

## Вывод (честно)

- btc_breakout на daily TradFi — **тупик по конструкции**, не по
  отсутствию рыночного edge. Ретрофитить крипто-15m-стратегию на
  дневной индекс бессмысленно (volume-семантика, диапазон цен, warmup
  — всё чужое).
- Подгонять параметры/окна, чтобы «получить хоть какие-то сделки» —
  запрещено дисциплиной (оверфит, урок XRP-скальп). Не делаем.
- Что действительно ценно из этой итерации — **инфраструктура данных**:
  теперь есть длинная история для честной разработки daily-стратегии.
