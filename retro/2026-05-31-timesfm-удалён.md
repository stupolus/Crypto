# Ретро 2026-05-31 — TimesFM удалён: не побил naive baseline

## Контекст

TimesFM v2.0 (500M Google foundation-модель прогноза временных рядов)
был добавлен как изолированная песочница в PR #179 (2026-05-26).
Использование в основной код проекта не интегрировалось — только
sandbox через `example.py`.

Владелец 2026-05-31: «не уверен в нём, потестируй, если будет
неэффективно — удали без жалости».

## Что сделал

`timesfm_integration/bakeoff.py` — честный walk-forward bake-off:
- Данные: BTC-USD daily с Yahoo (4275 баров, 2014-09 → 2026-05).
- Точки оценки: 30 out-of-sample прогнозов.
- Горизонт: 7 дней.
- Context: 512 баров.
- Метрика: MAE и MAPE.

Сравнение с тремя naive baselines:
- **Naive last-value** — постоянный прогноз = последняя цена.
- **Linear trend (30d)** — экстраполяция тренда по последним 30 точкам.
- **EMA(7) const** — EMA(7) от истории, констой вперёд.

## Результат (out-of-sample 30 точек × горизонт 7 дней)

| Метод | mean MAE | median MAE | mean MAPE % |
|---|---:|---:|---:|
| **TimesFM v2.0 500M** | 1720 | 1592 | 2.21 |
| **Naive last-value** | **1641** | **1389** | **2.10** |
| Linear trend (30d) | 2764 | 3280 | — |
| EMA(7) const | 2003 | 2008 | — |

### Skill TimesFM

- vs naive last-value: **−4.8%** (хуже!)
- vs linear trend: +37.8%

## Вердикт

**TimesFM проиграл naive last-value по out-of-sample MAE.** ~2 GB
модели, 50 слоёв, минута процессорного времени на forecast — **не
оправдывают себя** на BTC daily для нашего use-case.

Это согласуется с известной критикой foundation-моделей на финансовых
рядах: random walk + low signal-to-noise + fat tails — режим, где
massive pretraining не даёт edge'а над тривиальной экстраполяцией.

## Действие

- Запущен `timesfm_integration/uninstall.sh` → HF-кэш (~2 GB) удалён.
- `git rm -r timesfm_integration/` → весь sandbox удалён из репо.
- Зависимости (`torch`, `timesfm`, `jax[cpu]`) не трогали — они были
  только в локальном venv внутри папки, который удалился вместе с ней.

## Что бы могло сделать TimesFM полезным (для будущих экспериментов)

Если когда-то решим вернуться, **не** делать наивный price-forecast.
Возможные варианты, где foundation-time-series может дать что-то:

1. **Длинные горизонты** (60-90 дней). Naive last-value на больших
   горизонтах хуже из-за trending behavior; TimesFM с контекстом 512+
   может уловить mean-reversion на месяцах. Но для нашего use-case
   (intraday/daily entries) горизонт 60d бесполезен.
2. **Forecasting derived features** — не цены, а volatility, OI,
   funding rate, volume — могут быть менее «random-walk-y».
3. **Sentiment-driven datasets** где есть структурные циклы (например
   weekly seasonality). На крипте 24/7 weekly cycle слабый.
4. **TimesFM v3 / больше параметров** если выйдет с улучшенным
   pretraining на финансовых данных.

До тех пор — не возвращаемся, не теряем 2 GB диска и минуты на
forecast, который проигрывает constant.

## Урок

Любая «foundation-модель из коробки» должна проходить тот же гейт,
что и стратегии: **out-of-sample vs naive baseline**. Если не побила
naive last-value на чистом ценовом ряду — выкидывать без жалости. Это
не вопрос «может в каком-то use-case она лучше» — это вопрос «нашли
ли мы конкретный use-case, где она лучше». На BTC daily 7d — нет.
