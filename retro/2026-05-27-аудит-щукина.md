# Retro 2026-05-27 — Аудит Щукина (liquidation_reversal): провал критики

Запрос пользователя: «стратегия Щукина не выдерживает критики, проведи
аудит и добавь изменения». Аудитирую методологию из
`бизнес/материалы/каналы/dmitry-shukin-crypto.md` + плана 21
(`liquidation_reversal`) — без новых гипотез, по дереву goal-команды.

## Критические замечания

### 1. Методология никогда не валидирована
Plan 21 фаза 21.4 = «2-летний IS/OOS WF на 4h/15m liq+OI+CVD» —
**не выполнена**. Coinglass-тариф пользователя (HOBBYIST/$29 эквивалент)
в плане 32 показал: интервал ≥ 4h, история ~6 мес, 429 — полноценный
2-летний WF невозможен. Strategy в репо живёт как **безопасный no-op**
(StaticProviders → пустые ответы → 0 ордеров) — никто никогда не
видел её реальный edge.

### 2. «Числа из видео, не подгоняем под бэктест» — подмена дисциплины
Параметры (`level_n=50`, `liq_spike_min=5`, `liq_min_baseline_usd=100k`,
`cycle_wait=2`, `oi_lookback=6`, `tp1_r=3.0`) заявлены как
«из бизнес/материалы/курсы/dmitry-shukin/», но эта папка
**не существует** в репо (`ls бизнес/материалы/курсы/dmitry-shukin/`
→ нет). Канал doc сам говорит «🟡 Не обработано». То есть числа —
**из общей памяти**, и при этом «защищены от тюнинга» лозунгом
pre-registration. Это анти-дисциплина: pre-registration защищает от
p-hacking когда числа имеют независимое обоснование; «автор курса так
сказал» — не обоснование, особенно когда автор канал 229k с маркетинг-
следом (channel doc сам предупреждает: «229k подписчиков ≠
доказательство прибыльности»).

### 3. Жёсткий gate «не шортить пока OI растёт» — догма категории ⚠️
Сетап A2 требует `detect_oi_trend.state == FALLING` (бинарный gate).
Это утверждение типа «всегда/никогда» — собственная инструкция channel
doc относит такое в категорию ⚠️ «догма без доказательств, в
`что-проверить.md` или выкинуть, если не подтвердилось». Эмпирического
подтверждения нет.

### 4. Эмпирический косвенный негатив через composite_signal
`composite_signal` v1/v2 = инстанциация той же Щукин-композиции
(funding_extreme + liquidation_sweep + OI-gate, ± CVD). На реальных
Coinglass-данных (retro 2026-05-18 cvd-3-й сигнал-валидация):

| вариант | OOS PF (BTC 4h) | OOS PnL |
|---|---|---|
| v1 (2-of-3 без CVD) | 0.0 (2 сделки, вырожденно) | +4.2% (мусор) |
| v2 (3 сигнала + CVD) | **0.49** | **−4.2%** |

Не «не работает» доказательно (sample мал), но **точно не
подтверждает edge** на тестируемой части методологии.

### 5. R:R = 3.0 «канон курса» — неоткалиброванная константа
`tp1_r_multiple: 3.0` без обоснования относительно структуры стопов и
microstructure target/stop ratio. Для сравнения: btc_breakout 1.5,
box_breakout 1.8, composite v1 1.5. Курсовая «канонiчная» цифра — не
доказательство правильности 3.0.

### 6. Источниковые материалы отсутствуют
Plan 21 ссылается на `бизнес/материалы/курсы/dmitry-shukin/{
основные-идеи.md, правила-входа.md, правила-выхода.md,
риск-менеджмент.md, что-проверить.md, что-отбросить.md}` — **ни одного
файла не существует**. То есть план опирается на «выжимку, которая
будет когда-нибудь сделана». Реальная база = устные тезисы.

## Применённые изменения (этот PR)

1. **Этот retro** — фиксирует аудит.
2. **plan 21** — addendum со статусом «AUDIT-DEPRECATED 2026-05-27».
3. **channel doc** — критическая секция с findings, статус
   «методология не подтверждена, в live не идёт».
4. **`strategies/liquidation_reversal/strategy.py`** — docstring-блок
   с явной пометкой audit-deprecated и ссылкой на этот retro.
5. **`runners/live_runner.py` choices** — `liquidation_reversal`
   удалён из списка выбираемых стратегий. Код + тесты остаются как
   историческое (можно вернуть отдельным PR при появлении валидных
   данных), но из CLI больше не выбирается случайно.
6. **`scripts/run_backtest.py` choices** — то же.

## Что НЕ трогаю

- **composite_signal forward-демо** — продолжает на VPS по
  kill-критерию. Это эмпирическая проверка той же Щукин-композиции;
  scope-команда заморозила параметры демо, изменения «по ходу»
  запрещены. Пусть данные решат через kill.
- Код стратегии (strategy.py логика) и его тесты — оставляю как
  исторический артефакт. Уничтожать рабочий код не нужно; достаточно
  отключить выбор в раннере.

## Вердикт

Методология Щукина в текущей реализации (`liquidation_reversal`) —
**не подтверждена ни теоретически, ни эмпирически**. Деактивирована в
раннерах. NO-GO держится — это не противоречит общему вердикту, а
конкретизирует одну из ветвей.
