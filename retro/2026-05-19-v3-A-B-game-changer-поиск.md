# Retro 2026-05-19 — composite v3 + планы A/B + честная проверка info-edge

## Контекст

Запрос: «улучшить ещё, должен появиться game-changer, всё, что
считаешь нужным». Прямо ответил, что **game-changer через ещё одну
крутилку composite на 4–7 OOS-сделках = подгонка**, и это запрещено.
Сделано три вещи параллельно: v3 (одно теоретическое улучшение), план
A (под апгрейд тарифа), план B + реальный probe.

## v3 — одно улучшение, не пять (план 44)

**Session-time gate** (`session_hours_utc`): не торгуем в азиатский
low-liq period; контртрендовые funding/liq-setup'ы кластеризуются в
US/EU-овлапе (теория микроструктуры). Opt-in, default None = поведение
v2. config-v3.yaml: 13..21 UTC.

Демо v1 байт-в-байт прежний (доказано `test_v2_defaults_preserve_v1`
+ 11 прежних composite-тестов зелёные). v3 на live НЕ идёт — тот же
gate iter#4 (плана 45 после апгрейда тарифа).

**v3↔v2 на текущих данных НЕ прогонялся** — на 4 OOS-сделках любой
вердикт = шум, не доказательство (явно зафиксировано в плане 44).

## План 45 (A) — WF-протокол под апгрейд Coinglass

Pre-registered: 15m + ≥12 мес + WF (IS=180 OOS=60 step=60, ≥9 окон,
≥30 сделок/версия) + критерий iter#4. Гейт на single-вариант: должен
проходить на BTC И ETH. v2/v3 — кандидаты, не «улучшения по факту».

## План 46 (B) — info-edge: macro lead-lag probe (выполнен)

Pre-registered (до прогона): H0 = DXY/VIX не опережают BTC на лагах
1..3. Два теста (lag-corr + conditional-mean по децилям), permutation
p-value, Bonferroni α/12 = 0.00417.

Прогон: 2105 совпавших дней (2018-2026), Yahoo (свободно). Результат:

```
Lag-correlation:    все p ∈ [0.04, 0.79], мин 0.044 (DXY k=3)
Conditional mean:   все p ∈ [0.16, 0.84]
```

**Ни один из 12 тестов не прошёл Bonferroni-порог.** На 8 годах
данных это не «мало», это **отклонение H1**: lead-edge DXY/VIX → BTC
не подтверждается. Не дотягиваем («давайте 7 лагов вместо 3») — это
было бы p-hacking, запрещено самим планом 46.

⇒ Macro-гипотеза закрыта. Composite **не получает** 4-го info-голоса
из macro. Это game-NOT-changer, но честный.
