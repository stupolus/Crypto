# Backlog идей

Все «хорошо звучит, но потом» из чатов. Каждая идея — с обоснованием, когда возвращаться.

## Принцип

Идеи сюда складываются БЕЗ принятия решения «делаем». Делаем, только когда:
1. Текущая фаза стабильна.
2. Есть гипотеза, как идея даст измеримый edge.
3. Есть способ проверить на бэктесте или статистике.

## Источники данных (отдельные стратегии не делаем)

### Polymarket как источник макро-вероятностей
**Что:** парсер цен ключевых рынков (ставки ФРС, рецессия, BTC-таргеты).
**Зачем:** контекстный фильтр для нашей торговли (например, «ФРС снизит» → бычий контекст для золота).
**НЕ делаем:** торговать на Polymarket. Другая природа эджа, отдельный проект.
**Когда:** фаза 3+, когда основные стратегии работают.

### Whale wallet tracker
**Что:** список 10–20 публичных кошельков известных трейдеров (Hyperliquid + Ethereum), мониторинг входов/выходов.
**Зачем:** контекстный сигнал «крупный игрок зашёл в позицию» — НЕ копирование, а фон.
**НЕ делаем:** автокопию. Слишком много шума, разные горизонты, разные хеджи.
**Когда:** фаза 3+.

### Twitter/X-парсер
**Что:** мониторинг ключевых аккаунтов (Маск, политики, главы ФРС).
**Зачем:** фильтр «не входим против горячей новости».
**Барьер:** Twitter API дорогой, ROI спорный. На fase 5+ если вообще.

## Стратегии

### Mean reversion после ликвидаций
**Что:** вход в обратную сторону после крупного каскада ликвидаций (например, > $5M на BTC).
**Зачем:** документированный паттерн mean reversion (30мин–2ч окно).
**Когда:** фаза 2, после стабильного breakout-бота.

### Volatility breakout (открытие сессий)
**Что:** вход на пробой ATR-канала на открытии важных сессий (Asia open, US open).
**Зачем:** дополнительный класс сетапов, добавляет к моментум-стратегии.
**Когда:** фаза 3+, как третья стратегия.

### News filter
**Что:** не «торгуем по новостям», а «не торгуем 30 мин до и 30 мин после значимой новости».
**Источники:** Forex Factory calendar (бесплатный), Cryptopanic API.
**Зачем:** убирает половину «непонятных» убытков от новостных свечей.
**Когда:** фаза 4, как фильтр поверх существующих стратегий.

### Funding extremes (контр-трендовая)
**Что:** при экстремальном funding (>0.1%/8ч или >0.05%/4ч) ищем разворот.
**Зачем:** "когда все стоят на одной стороне, рынок поворачивается" — измеримый эффект.
**Когда:** фаза 1–2, можно начать собирать данные сразу.

## ML / автоматизация

### ML-фильтр поверх rule-based
**Что:** простая модель (logistic regression / gradient boosting) на признаках (волатильность, тренд, время, RSI, ATR), отсеивает заведомо плохие сетапы.
**НЕ делаем:** ML как замену стратегии. Только как фильтр на топе.
**Когда:** фаза 5+, после 200+ сделок в журнале + стабильной базовой стратегии.

### Auto-экстракция правил из журнала
**Что:** Claude раз в месяц анализирует журнал сделок, ищет статистические закономерности, предлагает новые правила.
**Это НЕ ML.** Это data analysis + human iteration.
**Когда:** делаем сразу, как только наберётся 50+ сделок.

## Инфраструктура

### Hyperliquid native data feed
**Что:** прямое подключение к Hyperliquid WS для real-time ликвидаций, OI, funding (бесплатно, on-chain).
**Зачем:** Hyperliquid = ~52% RWA-перпов, основной источник для золота/индексов.
**Когда:** фаза 1 (читаем как данные), фаза 2 (используем в стратегии).

### Coinglass API (heatmap)
**Что:** карта ликвидаций по всем основным парам.
**Зачем:** TP placement, stop placement, setup confirmation.
**Стоимость:** ~$29/мес.
**Когда:** фаза 1 (после первого живого месяца на BTC).

### Telegram-алерты
**Что:** бот шлёт уведомления о критичных событиях (стоп бота, exit-сделка, аномалия).
**Когда:** фаза 0, обязательно.

### Дашборд
**Что:** веб-страница с PnL, открытыми позициями, статистикой за день/неделю/месяц.
**Стек:** Next.js + Recharts, или Grafana, или streamlit (быстрее).
**Когда:** фаза 1 (после testnet).

## Образовательные источники для обработки

### Курсы Дмитрия Щукина
**Что:** материалы лекций (видео).
**Как обработать:** Whisper → транскрипт → выжимка в `материалы/курсы/dmitry-shukin/`.
**Принцип:** скептически, делим на 4 ведра (см. CLAUDE.md / структура обработки).

### Канал Никиты Ануфриева "Хедлайнеры"
**Что:** макро-наблюдения, рыночные доли, аномалии funding.
**Как использовать:** еженедельный обзор, ключевые наблюдения → `материалы/выжимки/`.

### Книги по трейдингу
**Что:** список TODO (заполнить).
**Как обработать:** PDF → выжимка в `материалы/книги/<название>.md`.

## Разное

### Carbon footprint API ключей
**Что:** мониторинг утечки/компрометации API ключей (например, через keysearch на github).
**Когда:** фаза 2+, когда есть live-ключи.

### Volumetric backtest (а не tick)
**Что:** при бэктесте учитывать ликвидность стакана, а не идеальное исполнение.
**Зачем:** реалистичные результаты бэктеста, без «бумажной» прибыли.
**Когда:** фаза 1 для базового, фаза 2 для расширенного.

## Решительные «нет»

### ❌ Скальпинг с большим плечом
Никогда. См. [[правила-торговли/анти-скальпинг]].

### ❌ Копирование чужих сделок (TG-сигналы как триггер)
Чужие сигналы = данные, не входы. Каждый источник проходит статистическую проверку.

### ❌ Мартингейл, усреднение в убыток, "Grid bots без верха"
Математически проигрывают на тренде. Не делаем.

### ❌ Front-running на mempool
Юридически серая зона + конкуренция с MEV-ботами. Не для нас.

### ❌ Hyperliquid как основная биржа исполнения
Слишком молодая, JELLY-инцидент показал, что биржа может вмешаться. Используем как **источник данных**, не как venue.
