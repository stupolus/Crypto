# Выжимка: 13 YouTube-роликов «Claude Code + трейдинг»

Источник: ветка `claude/yt-extra-youtube-trading` (PR #151), 13 сырых
ASR-транскриптов в `транскрипты/youtube-trading/`. Тема — не торговая
теория, а мета-туториалы «подключи Claude Code к TradingView / собери
бота за 14 минут / я дал Claude $200k».

## Вердикт по 4 вёдрам

- ✅ **Тестируемых правил: НЕТ.** Ни одного параметризованного,
  доказанного сетапа. Все «+530%», «+112%», «35 944%» — либо открыто
  grid-overfit (08), либо сломанный бэктест (12 — налив по неверной
  свече, −$11k), либо без правил вообще. По методологии скептицизма:
  «229k подписчиков ≠ прибыльность», «+X% без правил → ❌».
- ❌ **Маркетинг/мотивация (большинство):** 01, 03, 05, 07, 08, 12, 13 —
  affiliate/funnel (Telegram-PDF, WhatsApp-группа, Hostinger, lighter,
  «inner circle»). Игнор.
- ⚠️ **Догма:** 10 — brute-force оптимизатор как «грааль» (сам автор
  потом признаёт просадку/оверфит).
- 🟡 **Контекст/инфраструктура (единственное полезное):** см. ниже.

## 🟡 Что забрать (только инфра/дисциплина, НЕ стратегии)

1. **Анти-look-ahead в режимных моделях (06).** При HMM/любой
   разметке режимов использовать forward-only оценку, не «обучил на
   всём ряду → predict на нём же». Требовать персистентности режима
   ≥3 баров перед действием. Самая ценная техническая заметка из всех 13.
2. **Хардкод circuit breakers, независимые от ИИ (06).** −2%/день →
   половина сайза; −3% → flat; −5%/нед → половина; −10% от пика →
   файл-блокировка, снимается только вручную. Паттерн «файл-блок,
   который надо удалить руками» — кандидат в `core/` риск + `ops/`.
   Согласуется с уже действующими лимитами CLAUDE.md (−3%/−7%/−15%).
3. **Класс бага бэктеста для теста-ассерта (12).** Налив сделки на
   неверной свече (сигнал по цене X, исполнение по X+N). Добавить
   регресс-тест в бэктест-движок: исполнение строго по бару сигнала/
   следующему open, не по будущему экстремуму.
4. **Ограничение источников данных (02/04).** Yahoo Finance — только
   daily, для интрадей непригоден (подтверждает наши прежние замеры).
   TradingView Desktop+MCP может отдавать суб-минутные OHLCV — но это
   их платформа; мы торгуем BingX/Bybit, как источник данных не нужен.
5. **Read-only мульти-агент аналитики (09).** Шаблон: отдельные
   агенты tech/fundamental/sentiment/risk → синтез тезиса. Не ново,
   но совпадает с Принципом 1 (источник = фича/фильтр, не триггер).

## Итог

Торгового edge в материале нет. Ценность чисто методическая (пункты
1–3). Сырые транскрипты как артефакт ценности не несут — рекомендуется
закрыть PR #151 и удалить ветку `claude/yt-extra-youtube-trading`
(только с подтверждением пользователя). Эта выжимка сохраняет всё
полезное; перечитывать 43k слов хайпа не нужно.
