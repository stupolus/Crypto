# Риск-профиль

## Базовые инварианты

| Параметр | Значение | Почему именно так |
|---|---|---|
| **Риск на стандартную сделку (B-tier)** | 1.0% эквити | Классика R-модели. При winrate 55% и среднем 1.5R — устойчивый рост. |
| **Риск на A-tier сделку** | 1.5% эквити | Сильный сетап заслуживает больше, но не катастрофического риска |
| **Риск на A+-tier сделку** | 2.0% эквити | Максимум для одной сделки. Никогда выше |
| **Дневной стоп-лосс** | −3.0% эквити | После 3 убыточных сделок по 1% бот выключается до завтра |
| **Недельный лимит** | −7.0% эквити | Пауза на неделю, обязательное retro |
| **Месячный лимит** | −15.0% эквити | Стоп проекта, разбор полётов, возможен пересмотр стратегий |
| **Макс. одновременных позиций (на старте)** | 1 | Один инструмент = одна позиция. Без пирамидинга |
| **Макс. сделок в день** | 3 | После 3 убыточных подряд — стоп до завтра, даже если дневной лимит не превышен |
| **Макс. эффективное плечо (B-tier)** | 3x | Достаточно для большинства сетапов |
| **Макс. эффективное плечо (A+ tier)** | 5x | Только при подтверждённом сетапе |
| **Абсолютный потолок плеча** | 5x | НИКОГДА выше. Биржевое плечо на ползунке может быть выше, но размер позиции считается от риска |

## Формула размера позиции

```
notional = (эквити × риск_на_сделку%) / расстояние_до_стопа%
position_size = notional / цена_входа
effective_leverage = notional / эквити
```

**Перед каждым ордером проверяем:**
- `effective_leverage <= 5` — иначе ордер не ставится
- `(цена_ликвидации − цена_стопа) >= 30% от расстояния_до_стопа` — стоп должен срабатывать сильно раньше ликвидации

## Примеры расчёта

Депозит **$1 000**, риск **1%** = $10 на сделку.

| Стоп от входа | Notional | Эфф. плечо | Что при стопе |
|---|---|---|---|
| 0.5% | $2 000 | 2x | −$10 (1% эквити) |
| 1.0% | $1 000 | 1x | −$10 |
| 1.5% | $667 | 0.67x | −$10 |
| 2.0% | $500 | 0.5x | −$10 |

Депозит **$10 000**, риск **1%** = $100 на сделку. Числа умножаются.

## Маржинальный режим

- **Изолированная маржа всегда.** Кросс — никогда (один убыточный трейд не должен потянуть весь счёт).
- **Stop-loss ставится одновременно с входом**, не «потом». Если биржа не успела зафиксировать стоп — позиция закрывается рыночным.

## Жёсткие запреты

- ❌ Плечо > 5x — в любых обстоятельствах.
- ❌ Стоп тоньше 0.5% — на крипте это уровень шума, всегда выносит.
- ❌ Скальпинг (вход на цель < 0.5%) — комиссии и слиппедж съедают edge. См. [[правила-торговли/анти-скальпинг]].
- ❌ Усреднение в убыток — никогда. Если сетап не работает, выходим, не «доливаем».
- ❌ Нет стопа на бирже — нет позиции. Пока стоп не подтверждён биржей, бот не считает позицию открытой.
- ❌ Кросс-маржа — никогда.

## Циркуит-брейкеры (автоматические)

```
дневной_PnL <= -3% эквити        → бот: STOP до завтра
3 убытка подряд                  → бот: STOP до завтра
недельный_PnL <= -7% эквити      → бот: STOP до понедельника + alert
месячный_PnL <= -15% эквити      → бот: STOP, ручной перезапуск только после retro
WS-отвал > 30 сек                → закрываем все позиции, ждём восстановления
ack ордера > 5 сек               → флагуем, но не закрываем
volatility за 5 мин > 5%         → пауза на 30 мин (паника / новость)
```

## Источники / основания

- R-модель: классика "Trade Your Way to Financial Freedom" (Van Tharp).
- 1% per trade: множество исследований fund-managment, Kelly criterion с consistency factor.
- Daily/weekly/monthly stops: эмпирика prop-firms (FTMO, Topstep — у них похожие лимиты).
- Анти-скальпинг: математика комиссий + волатильность крипты.
