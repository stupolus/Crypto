## Базовые инварианты

Профиль смелее прежнего: риск/сделку и потолок плеча подняты, добавлен отдельный `SCALP` tier, лимиты дня/недели/месяца пересчитаны под бóльшую частоту и риск. Жёсткие тормоза (изолированная маржа, стоп с входом, буфер до ликвидации, циркуит-брейкеры, запрет 25x) остаются.

> **Статус внедрения (план 22 — реализовано):** числа ниже зеркалятся в `core/risk/config.yaml`, `RiskEngine` их enforce'ит. Свинговые tier'ы (B/A/A+) и новые лимиты/брейкеры — активны. `SCALP` tier за **backtest-gate**: `scalp.enabled=false` по умолчанию → движок отклоняет скальп (`SCALP_DISABLED`), пока скальп не подтверждён бэктестом (фаза 22.4) и не включён явным «да». Меняем md → потом config, никогда наоборот.
>
> Потолок эфф. плеча движок enforce'ит как **единый абсолютный** `max_effective_leverage=10x` (жёсткий инвариант). Разбивка «B/A 7x» ниже — ориентир дисциплины, отдельным per-tier гейтом пока не enforced (вынесено как возможное уточнение плана 22).

| Параметр | Значение | Почему именно так |
|---|---|---|
| **Риск на SCALP-сделку** | 0.75% эквити | Высокая частота → мелкий риск на штуку держит дневную просадку в рамках |
| **Риск на стандартную сделку (B-tier)** | 1.5% эквити | Смелее прежнего 1.0%. R-модель устойчива при winrate 55%+ |
| **Риск на A-tier сделку** | 2.0% эквити | Подтверждённый сетап заслуживает больше |
| **Риск на A+-tier сделку** | 3.0% эквити | Максимум для одной сделки. Никогда выше |
| **Дневной стоп-лосс** | −5.0% эквити | Пересчитан под бóльший риск/сделку. Стоп до завтра |
| **Недельный лимит** | −10.0% эквити | Пауза на неделю, обязательное retro |
| **Месячный лимит** | −20.0% эквити | Стоп проекта, разбор полётов, возможен пересмотр стратегий |
| **Макс. одновременных позиций** | 3 (correlation-adjusted) | Больше параллели; нетто-экспозиция считается через корреляции |

## Формула размера позиции

```
notional = (эквити × риск_на_сделку%) / расстояние_до_стопа%
position_size = notional / цена_входа
effective_leverage = notional / эквити
```

**Перед каждым ордером проверяем:**
- `effective_leverage <= 10` — иначе ордер не ставится (абсолютный потолок)
- `(цена_ликвидации − цена_стопа) >= 30% от расстояния_до_стопа` — стоп должен срабатывать сильно раньше ликвидации
- **Пред-входная оценка ликвидации** (изолированная маржа, linear USDT-перп): `liq_LONG ≈ entry·(1 − 1/L + mmr)`, `liq_SHORT ≈ entry·(1 + 1/L − mmr)`. `mmr` (maintenance margin rate) `= 0.01` — консервативно выше tier-1 ставки BingX по мажорам (запас в безопасную сторону: завышенный mmr двигает оценку liq ближе к entry → буфер-чек строже). `L` для оценки берётся = потолок плеча тира (фактическое risk-based плечо ниже → реальная liq дальше → оценка пессимистична). Цифра — `core/risk/config.yaml: limits.maintenance_margin_rate`.
- **SCALP только:** `TP% >= k × trade_cost%` (fee-edge-gate, `k ≥ 2`, `trade_cost ≈ fee + slippage`). Скальп, не отбивающий трение, отклоняется. См. [[правила-торговли/правила-скальпинга]]

## Примеры расчёта

Депозит **$1 000**, риск **1%** = $10 на сделку.

| Стоп от входа | Notional | Эфф. плечо | Что при стопе |
|---|---|---|---|
| 0.5% | $2 000 | 2x | −$10 (1% эквити) |
| 1.0% | $1 000 | 1x | −$10 |
| 1.5% | $667 | 0.67x | −$10 |
| 2.0% | $500 | 0.5x | −$10 |

Депозит **$10 000**, риск **1%** = $100 на сделку. Числа умножаются.

## Маржинальный режим

- **Изолированная маржа всегда.** Кросс — никогда (один убыточный трейд не должен потянуть весь счёт).
- **Stop-loss ставится одновременно с входом**, не «потом». Если биржа не успела зафиксировать стоп — позиция закрывается рыночным.

## Жёсткие запреты

- ❌ Эфф. плечо > 10x — в любых обстоятельствах. 25x-казино — навсегда нет.
- ❌ Стоп тоньше 0.15% — даже на скальпе это уровень шума.
- ❌ Скальп без fee-edge-gate (TP не покрывает трение) — отклоняется движком. См. [[правила-торговли/правила-скальпинга]].
- ❌ Усреднение в убыток — никогда. Если сетап не работает, выходим, не «доливаем».
- ❌ Нет стопа на бирже — нет позиции. Пока стоп не подтверждён биржей, бот не считает позицию открытой.
- ❌ Кросс-маржа — никогда.

## Циркуит-брейкеры (автоматические)

```
дневной_PnL <= -5% эквити        → бот: STOP до завтра
5 убытков подряд                 → бот: STOP до завтра
25 SCALP-сделок за день          → SCALP-вход: STOP до завтра (свинг продолжает)
недельный_PnL <= -10% эквити     → бот: STOP до понедельника + alert
месячный_PnL <= -20% эквити      → бот: STOP, ручной перезапуск только после retro
WS-отвал > 30 сек                → закрываем все позиции, ждём восстановления
ack ордера > 5 сек               → флагуем, но не закрываем
volatility за 5 мин > 5%         → пауза на 30 мин (паника / новость)
```

## Источники / основания

- R-модель: классика "Trade Your Way to Financial Freedom" (Van Tharp).
- 1% per trade: множество исследований fund-managment, Kelly criterion с consistency factor.
- Daily/weekly/monthly stops: эмпирика prop-firms (FTMO, Topstep), пересчитаны под бóльший риск/сделку.
- Скальп через fee-edge-gate: математика комиссий + волатильность крипты (рамка, не запрет).
