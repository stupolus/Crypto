# Как добавлять материалы (инструкция)

Это самое простое объяснение «куда что класть, что Claude сделает». Если устал читать длинный план — читай только этот файл.

## Главный принцип

Ты кидаешь файл в нужную папку `бизнес/inbox/<тип>/`. В новой Claude-сессии говоришь: **«разбери inbox»**. Claude сам читает, разбирает, складывает выжимку в `бизнес/материалы/обработанное/`, делает коммит.

Сырой файл (видео, большой PDF, скрин) **в гитхаб не попадает** — он только у тебя локально. В репо едут только текстовые выжимки.

## Куда что класть

| Что у тебя | Папка | Пример |
|---|---|---|
| PDF — книга, отчёт, статья в PDF | `бизнес/inbox/pdf/` | `книга-tharp.pdf` |
| Видео (mp4/mkv/mov) — лекция, ролик | `бизнес/inbox/video/` | `shukin-vsa.mp4` |
| Аудио (m4a/mp3/ogg) — голосовая, подкаст | `бизнес/inbox/audio/` | `мысли-2026-05-12.m4a` |
| Скриншот (png/jpg) — график, экран биржи, скрин TG | `бизнес/inbox/screenshots/` | `btc-15m-pump.png` |
| Текст (txt/md/html, экспорт TG в json) | `бизнес/inbox/text/` | `tg-export-headliners.json` |
| Просто URL | `бизнес/inbox/links/` | в `links.md` пишешь ссылки построчно |

Если не уверен куда — клади в `text/` и подпиши в имени файла что это. Claude разберётся.

## Как назвать файл

Никаких правил. Хоть `IMG_1234.png`, хоть `вот-что-я-думаю.m4a`. Claude переименует при обработке. Главное — **тип файла → правильная папка**.

## Как сказать Claude разобрать

В новой сессии (или прямо в текущей) — одна из фраз:

- **«Разбери inbox»** — пройдёт по всем папкам, обработает всё что найдёт.
- **«Разбери видео в inbox»** — только конкретный тип.
- **«Вот видео в inbox/video/shukin-vsa.mp4, сделай выжимку»** — точечно.

Что Claude сделает:

1. Прочитает файл. **PDF читаются только через markitdown** (PDF → Markdown), не «как есть» — это сохраняет заголовки/таблицы. Утилита: `python -m scripts.pdf_to_md <файл.pdf>` (зависимость: `pip install -e ".[materials]"`). Для аудио/видео — попросит прогнать через Whisper, если у тебя ещё не настроено локально.
2. Разложит по «4 ведрам»: ✅ тестируемое правило / 🟡 контекстная мудрость / ⚠️ догма без доказательств / ❌ маркетинг.
3. Создаст `.md`-выжимку в `бизнес/материалы/обработанное/<категория>/<имя>-<дата>.md` по подходящему шаблону из `бизнес/templates/`.
4. Поставит wiki-ссылки на `[[риск-профиль]]`, `[[идеи]]`, `[[материалы/каналы/...]]` где уместно.
5. Сырой файл оставит в inbox или переименует в `_processed_<имя>` — чтобы ты видел, что он уже разобран.
6. Сделает коммит в feature-ветку и запушит.

## Что Claude НЕ сделает

- ❌ Не выкинет твой сырой файл (только переименует в `_processed_`).
- ❌ Не запустит ничего на бирже (это не про этот пайплайн).
- ❌ Не закоммитит в гитхаб большие файлы (`бизнес/inbox/**` в `.gitignore`).
- ❌ Не закоммитит скрин с открытым API-ключом / email / балансом — обязан проверить и попросить тебя замазать.

## Безопасность скриншотов

Перед тем как кинуть скрин в `inbox/screenshots/`, **проверь что на нём не видно**:

- API-ключей (или их кусков).
- Email'ов (твоего и чужих).
- Реальных балансов в долларах.
- Имён пользователей в TG, если переписка не для всех.

Если что-то из этого на скрине — **закрась перед сохранением** (любой редактор картинок умеет). Если поленился — Claude обязан спросить и не коммитить.

## Шаблоны (для самого Obsidian)

В Obsidian есть встроенный плагин «Templates». Папка шаблонов уже настроена: `бизнес/templates/`. Есть 4 шаблона:

- `шаблон-видео-выжимка.md`
- `шаблон-статья-выжимка.md`
- `шаблон-торговая-идея.md`
- `шаблон-заметка-дня.md`

Когда хочешь сам быстро записать заметку или идею (не через inbox-конвейер):

1. Создай новый файл в нужной папке.
2. `Cmd/Ctrl+P` → «Insert template» → выбери шаблон.
3. Заполни. Если не уверен какой — `шаблон-заметка-дня.md` по умолчанию.

## Раздел для Claude (а не для пользователя)

Если ты Claude и читаешь это — заметки про работу с vault'ом:

- **PDF — всегда через markitdown.** Не читай PDF напрямую. Гони через `scripts/pdf_to_md.py` (`convert_pdf()` / `python -m scripts.pdf_to_md`). Единая точка чтения PDF — план `plans/53-markitdown-pdf-integration-2026-06-06.md`.
- **Wiki-links не авторефакторятся при правке через Edit/Write.** При переименовании или перемещении файла внутри vault'а — `grep -r "\[\[старое-имя\]\]"` по всему репо и обнови все ссылки в той же сессии. Не оставляй битых ссылок.
- **Не дублируй цифры из `риск-профиль.md` в выжимках.** Только `[[риск-профиль]]` ссылка. Иначе при изменении профиля выжимки рассинхронятся.
- **Если в материале появилась гипотеза, требующая бэктеста** — не пиши порог в код стратегии. Сначала в выжимку как «к проверке», потом в `идеи.md` как backlog-пункт, и только после бэктеста — в `plans/<имя>.md` как параметр.
- **Категории внутри `обработанное/`** создавай по мере появления первого файла категории: `видео/`, `статьи/`, `книги/`, `скрины/`, `аудио/`, `tg-экспорты/`, `заметки/`. Не плоди пустых папок наперёд.
- **После обработки inbox должен опуститься.** Либо файлы переименованы в `_processed_<имя>`, либо удалены (только по согласованию с пользователем).
- **Перед коммитом скринов из `обработанное/скрины/_assets/`** проверять размер. Лимит 500KB на скрин. Если больше — пережимать или хранить ссылкой, а не файлом.
