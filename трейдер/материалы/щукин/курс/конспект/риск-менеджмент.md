# Риск-менеджмент (видео 013 — таблица)

🟡/✅. Сверка с `бизнес/риск-профиль.md` и `core/risk/`.

## Заявленные правила курса

| Параметр | Курс Щукина | Наш риск-профиль | Вывод |
|----------|-------------|------------------|-------|
| Риск/сделку | 2% «общепринято», лично 1%, новичкам 0.5% | 1% (A+ до 2%) | ✅ совпадает |
| R:R | старт 1:3, мин 1:2, 1:1 запрещён | — (нет явного) | ⚠️ добавить в риск-профиль? |
| Размер позиции | = риск / дистанция_стопа | RiskEngine так и считает | ✅ совпадает |
| Плечо | «нерелевантно, важен объём позиции» | max 5x cap | ✅ согласуется |
| Дневной стоп | не назван явно | −3% эквити | наш строже — ок |

## Формула размера (подтверждает наш RiskEngine)

```
notional = (equity * risk_pct) / stop_distance_pct
```
Примеры из видео 013:
- риск $5, SL 0.4% → позиция ≈ $1250
- риск $5, SL 1.2% → позиция ≈ $416
- риск $5, SL 4.4% → позиция ≈ $115

→ `core/risk/engine.py` уже реализует ровно это. **Расхождений нет.**

## Что взять в код

1. R:R = 1:3 для TP1 (сейчас btc_breakout 1.5R — занижено vs курс).
   Кандидат: `tp1_r_multiple: 3.0` в config course-стратегии.
2. Structural SL (за экстремум окна) — у нас уже есть `_compute_stop`
   (low/high N свечей ИЛИ min 0.5%). Совместимо.
3. Риск 1% (tier B) — без изменений, совпадает.

## Что НЕ брать

- Усреднение убыточной позиции (видео 006) — против CLAUDE.md
  «анти-скальпинг» и риск-профиля. ❌
- Хеджирование убыточной (видео 007) — отложено, не для MVP.
- Скальп 2-6% тейки 10-15 мин (видео 005) — против CLAUDE.md
  (латенси убьёт интрадей-HF). Берём swing-вариант 1:3.
