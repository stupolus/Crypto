# HANDOFF — для других Claude-сессий (читать первым)

Этот репозиторий обрабатывается **несколькими сессиями параллельно**.
Чтобы не было конфликтов и дублей — обязательные правила:

## 1. Ветки: читать здесь, писать к себе

- Ветка `claude/crypto-video-transcripts-PnCMQ` — **рабочая ветка
  сессии-сборщика транскриптов**. В неё каждые ~10 минут идёт
  авто-push (новый урок = новый коммит).
- **НЕ пушь в `claude/crypto-video-transcripts-PnCMQ` из другой
  сессии** — твои коммиты будут конфликтовать с авто-push.
- Бери материалы отсюда (`git fetch` + `git checkout` + периодический
  `git pull`), а свою работу веди и пушь в **отдельную ветку**
  (например `claude/<своя-задача>-XXXX`).

## 2. Никакого дублирования файлов

- Транскрипты YouTube: `бизнес/материалы/обработанное/транскрипты/
  dmitry-shukin-crypto/` (+`INDEX.md`) — 100 шт. уже есть, не
  пересоздавать.
- Транскрипты видео-уроков курса: `бизнес/материалы/
  курс-криптограмотность/raw/уроки/NNN-<lid>.md` — пополняется
  автоматически. Не транскрибировать заново урок, файл которого уже
  есть.
- Выжимки по 4 вёдрам: `бизнес/материалы/курсы/dmitry-shukin/`
  (6 файлов) — дополнять существующие, не плодить копии.

## 4. Как забрать только новое и не переобрабатывать (для сессии-потребителя)

Источник правды о готовности = **наличие файла** (имя детерминировано:
`NNN-<lid>.md`). Не пересоздавать существующее.

Сессия-потребитель НЕ чекаутит и НЕ пушит эту ветку. Она:

```bash
# 1. Один раз: запомнить, что уже обработала (SHA или список файлов)
git fetch origin claude/crypto-video-transcripts-PnCMQ
LAST=$(cat .last_consumed_sha 2>/dev/null || echo "")

# 2. Список НОВЫХ/изменённых транскриптов с прошлого раза:
RANGE=${LAST:+$LAST..}origin/claude/crypto-video-transcripts-PnCMQ
git diff --name-only $RANGE -- \
  'бизнес/материалы/курс-криптограмотность/raw/уроки/' \
  'бизнес/материалы/обработанное/транскрипты/dmitry-shukin-crypto/'

# 3. Прочитать содержимое файла БЕЗ checkout:
git show origin/claude/crypto-video-transcripts-PnCMQ:путь/к/NNN-<lid>.md

# 4. После обработки — сохранить новый рубеж:
git rev-parse origin/claude/crypto-video-transcripts-PnCMQ > .last_consumed_sha
```

Идемпотентность на стороне потребителя: вести свой список уже
переваренных `lid`/`videoid` (или `.last_consumed_sha`). Файл уже в
списке → пропустить. Так ни один урок не обрабатывается дважды и
ничего не теряется между запусками.
