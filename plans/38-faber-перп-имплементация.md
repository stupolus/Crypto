# План 38 — Faber index-timing через BingX-перп (имплементация)

## Контекст

План 37: Faber 200-дн SMA-тайминг на индексе акций — ПЕРВЫЙ
за сессию робастный кандидат (S&P500 Sharpe 1.40, maxDD
−57%→−13%, +6/6 декад; NASDAQ100 1.37, +5/5; предзадано,
не скан, 40–56 лет). Edge = риск-менеджмент лонг-беты на
активе с реальной премией за риск. Это НЕ p-hacking.

НО бэктест 37 — на САМОМ индексе (Yahoo). Торговать можно
только через **BingX TradFi-перп** (NASDAQ100/SP500 есть в
юниверсе владельца). Перп ≠ индекс: funding, базис, гэпы
выходных/ночи, часы, издержки. Этот план — честная проверка,
переживает ли edge переход индекс→перп, и дисциплинированный
путь к demo (НЕ live).

## Жёсткая рамка

- Стратегия ЗАФИКСИРОВАНА (Faber 200SMA, long-or-cash), НЕ
  трогаем параметры — иначе подгонка. Перенос, не пере-поиск.
- Гейт/робастность не ослабляются. CLAUDE.md/AGENTS.md в силе.
- Live ЗАПРЕЩЁН в этом плане. Demo — отдельная фаза после
  честного прохождения перп-реализма. Деньги — только после
  demo ≥4 нед + явное «да» владельца (CLAUDE.md).
- Не дискреционная торговля. Сигнал механический.

## Фаза 38.2 — перп-реализм (бэктест на BingX-перпе)

- Скачать BingX-клайны перпов NASDAQ100/SP500 (1d; какая
  глубина есть — честно зафиксировать).
- Сравнить перп vs Yahoo-индекс: базис, поведение гэпов
  (выходные/закрытие US), стабильность.
- Прогнать ТОТ ЖЕ Faber 200SMA на перп-серии с реалистичными
  издержками: taker/maker switch + оценка funding за периоды
  «в лонге» (перп-funding платится, когда в позиции; в кэше —
  нет). Гэпы моделируются (вход/выход по next-open перпа).
- Сравнить с индексным результатом 37: edge сохраняется,
  деградирует или исчезает после перп-трения.

## Фаза 38.3 — вердикт

- Перп-Faber держит Sharpe>0.8, maxDD заметно < buy&hold,
  робастен по доступным под-периодам → **кандидат на demo**
  (отдельный план 39: demo-харнес, риск-сайзинг
  риск-профиль.md, ≥4 нед, метрики live-исполнения).
- Перп-трение убивает edge (короткая история / большой
  funding / гэпы) → честно зафиксировать «edge есть на
  индексе, но не торгуем перпом нашего типа» и закрыть.

## 10 причин провала (априорно, честно)

1. BingX TradFi-перп — короткая история (мало под-периодов).
2. Перп-funding на лонг-позиции съедает премию за риск.
3. Гэпы выходных/ночи (индекс закрыт, перп — нет) → стопы/
   проскальзывание хуже модели.
4. Базис перп-vs-индекс нестабилен → сигнал по close шумит.
5. Faber-сигнал считаем по индексу или перпу? (предзадаём:
   по перп-close, как реально торгуем — без look-ahead).
6. Ликвидность TradFi-перпа BingX тоньше → реальный спред.
7. Короткая история перпа = риск переноса (in-sample-подобно).
8. 1d-перп-данные BingX могут быть ≤ года → нет ≥8 корзин.
9. Налог/комиссии BingX TradFi могут отличаться от 0.05%.
10. Даже OK-бэктест перпа ≠ деньги: решает demo.

## Жёсткие стопы

- Короткая перп-история → честно сказать «недостаточно для
  вердикта», НЕ натягивать.
- Не live/не demo здесь. Деньги — отдельно, по правилам.
