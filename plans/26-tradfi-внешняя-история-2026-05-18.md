# План 26 — TradFi (S&P500 / акции): edge на внешней длинной истории

## Дата: 2026-05-18 · База: планы 23-25, диалог про BingX TradFi

## Зачем / исправление ошибки

Прошлая логика «упираемся в глубину истории BingX» была неверной
рамкой: BingX завёл синтетические TradFi-контракты недавно (~10 дней
15m, ~5-10 мес 1d), но **сам индекс/акция имеют десятилетия истории**.
Edge ищем на реальном инструменте по длинной внешней истории; BingX —
только площадка исполнения.

Yahoo Finance (`query1.finance.yahoo.com/v8/finance/chart`,
`period1=0&period2=now&interval=1d`) доступен и отдаёт:
- ^GSPC (S&P 500): **14 215** дневных баров с 1970
- AAPL: 11 449 с 1980 · NVDA: 6 872 с 1999 · ^NDX, TSLA, MSTR — аналогично

Это в разы выше порога iter#3 (≥9 окон / 12 мес). Ограничитель снят.

## Caveat (сохраняем честность)

- Внешняя история — **дневная**. Внутридневной длинной истории нет
  (Yahoo intraday ~60д). Стратегия — **daily-таймфрейм**.
- BingX-контракт = синтетика 24/7 со своим funding/гэпами «рынок
  закрыт». Edge валиден на underlying; tracking-расхождение с
  BingX-исполнением документируем и проверяем ОТДЕЛЬНО на короткой
  BingX-истории (её копим параллельно). До этого — не live.
- Параметры btc_breakout калиброваны под 15m крипту. На daily equities
  baseline-прогон как есть; подгонку параметров НЕ делаем в этой
  итерации (риск оверфита, урок XRP-скальп).

## Статус (2026-05-18) — ВЫПОЛНЕНО (с честным выводом)

Инфра доставлена и работает: `download_equity` + тесты, длинная история
(GSPC 14215 баров с 1970, AAPL 11449 с 1980, …). Но btc_breakout
структурно не ложится на daily equities (3 причины — см. retro), а
посчитанные IS-метрики edged не показывают (PF mean 0.52). Вывод:
нужна стратегия под daily-режим (отдельный план), не ретрофит крипто-15m.
Отчёт: `retro/2026-05-18-tradfi-внешняя-история.md`.

## Что делаем

### 26.1 — фетчер внешних данных
`scripts/download_equity.py`: Yahoo chart API → jsonl в формате `Kline`
(совместимо с `core.backtest`/`run_backtest`/`walk_forward`). Аргументы:
`--symbol` (^GSPC/AAPL/...), `--out`. Маппинг: timestamp→open_time_ms,
o/h/l/c/v. Сохраняем в `data/candles/<slug>-1d.jsonl`.
- Unit-тест на парсинг ответа (фикстура, без сети).
- ruff/mypy strict чисто.

### 26.2 — поддержка daily в btc_breakout
`config.py`: `timeframe` Literal +`1d` (минимально; warmup-математика
считает по числу баров, не по строке — безопасно).

### 26.3 — данные
Тянем ^GSPC, ^NDX, AAPL, NVDA, TSLA, MSTR (дневные, максимум истории).

### 26.4 — baseline + полный walk-forward
Для каждого символа:
- split-прогон (`run_backtest --split-fraction 0.6`) — быстрый срез.
- полный WF (`walk_forward.py --is-days 365 --oos-days 90
  --step-days 90` — на дневных барах окна в днях; даёт десятки окон на
  истории с 1970/1980).
Метрики IS/OOS PF, PnL, WR, maxDD, Sharpe, OOS+ доля.

### 26.5 — отчёт
`retro/2026-05-18-tradfi-внешняя-история.md`: таблица по символам,
вердикт по критерию edge (iter#4: OOS PF mean >1.5, PnL >+2%, OOS+
≥2/3), сравнение индексов vs отдельные акции, рекомендация.

## Definition of Done
- Фетчер + тест зелёные; ruff/format/mypy strict; pytest -q зелёный.
- ≥6 символов, полный WF (десятки окон) — отчёт с числами в `retro/`.
- Чёткий вердикт: на чём edge подтверждён / не подтверждён.
- Прод `core/risk/config.yaml` не тронут. Live-решение — только после
  прохождения критерия + отдельной проверки BingX-tracking.
