# План 22 — Мульти-скрин активов и источники данных

## Проблема (из диалога 2026-05-15)

Пользователь справедливо спросил: почему бот не скринит много монет на
фьючах, не подключает доп. индикаторы Coinglass, TG-скринеры, новости,
DEX, Hyperliquid. Ответ из дисциплины проекта:

1. **Чужой сигнал ≠ триггер** (CLAUDE.md принцип №1). TG-скринеры,
   новости, DEX идут путём: парсим → копим статистику → проверяем
   edge → встраиваем как фильтр/фичу. Никогда как вход.
2. **Узкое место — не число монет, а edge + глубина выборки.**
   Скрин 2026-05-15 (7 монет, 2 ценовые стратегии): robust edge нет.
   `btc_breakout`/15m прибылен только на BTC (OOS PF 1.74, но 21
   сделка < порога 30) и случайно XRP — оверфит/шум. `trend_ema_4h`
   убыточен везде, 3–20 сделок (4h+360д → невалидно).
   Глубина данных: 15m ~5.5 мес, 4h ~360 д — мало для значимости.

Критерий приёмки edge (план 20): PF > 1.3 И Sharpe > 0.8 И ≥30 сделок
OOS, без подгонки параметров.

## Источники данных (предложение, в т.ч. неизвестные пользователю)

Помечено ★ — то, что пользователь не называл, но нужно.

1. **Coinglass HOBBYIST доп.**: long/short account ratio,
   top-trader position ratio, aggregated funding, OI by exchange.
   ★ liquidation heatmap — на старшем тарифе (после апгрейда).
2. ★ **Биржевой нативный API (BingX/Binance)**: OI, funding,
   basis/premium index в реалтайме — глубже и быстрее Coinglass
   для live-фильтров. Бесплатно, ключ уже есть.
3. ★ **Hyperliquid public API**: OI/funding/L2-стакан по всем
   перпам без ключа. Юзер уже торгует HYPE — прямой смысл.
4. ★ **Deribit DVOL** (implied vol BTC/ETH) + put/call skew —
   опережающий индикатор смены режима волатильности.
5. ★ **Stablecoin mint/burn** (USDT/USDC эмиссия) — макро-режим
   ликвидности крипты, опережающий сигнал притока денег.
6. ★ **DEX (DexScreener/Birdeye)**: всплеск объёма новых
   листингов — ранний моментум (фича аномалий, не триггер).
7. **TG-скринеры пользователя** + ★ новостные RSS (CoinDesk/
   The Block) — парсинг → статистика → фильтр после edge-проверки.
8. ★ **Whale-flow** (крупные ончейн-переводы на биржи) — фича
   давления на цену.

Каждый источник: parse → store → backtest edge → решение
(фильтр/фича/отбросить), фиксируется здесь.

## Принципы (жёстко)
- Никакой подгонки под бэктест. Параметр обоснован логикой.
- Чужой сигнал — фильтр/фича после статистики, не триггер.
- Каждая фаза = коммит. Закрытая тема = PR.
- Деплой в Манус — только после подтверждённого edge, явным флагом.
