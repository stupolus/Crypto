## 2. Цель плана

`core/signals/` + `strategies/btc_breakout/`:

1. **`core/signals/indicators.py`** — чистые функции:
   - `donchian_channel(candles, n)` → `(upper, lower)`.
   - `atr(candles, period)` → Decimal.
   - `sma(values, period)` → Decimal.
   - `ema(values, period)` → Decimal (Wilder smoothing, как ATR).
   - `percentile(values, fraction)` → Decimal (для ATR-percentile фильтра).
2. **`core/signals/composite.py`** — провайдеры внешнего контекста:
   - `FundingProvider` protocol: `get_funding_rate(symbol, ts) -> Decimal | None`.
   - `NewsCalendar` protocol: `is_paused(ts) -> bool`.
   - `Blacklist` protocol: `contains(symbol) -> bool`.
   - Простые in-memory реализации для тестов и MVP.
3. **`strategies/btc_breakout/strategy.py`** — `BtcBreakoutStrategy`:
   - Реализует `Strategy` protocol из backtest.
   - State machine: `FLAT → PENDING → OPEN → TP1_HIT → CLOSED`.
   - После TP1: trailing exit по `close < EMA(21)` (LONG) / `close > EMA(21)` (SHORT).
   - Внутри использует RiskEngine для расчёта размера.
4. **`strategies/btc_breakout/config.yaml`** — все параметры из плана 08 §5.

**Что НЕ делаем:**
- ❌ Скрипт загрузки исторических свечей (`download.py`) — отдельная задача после прогона.
- ❌ TP2 как отдельный ордер на бирже (мы используем trailing exit через market reduce_only).
- ❌ Reduce-50%-at-TP1 на бирже как отдельный order — на MVP закрываем 50% market reduce_only при касании TP1 (backtester эмулирует так же).

## 3. Спецификация

### 3.1 Indicators

```python
def donchian_channel(candles: Sequence[Kline], n: int) -> tuple[Decimal, Decimal]:
    """upper = max(high), lower = min(low) за последние n свечей.
    Использует свечи [-n:], не включая текущую (стратегия передаёт history[:-1])."""

def atr(candles: Sequence[Kline], period: int) -> Decimal:
    """Wilder ATR. True Range = max(high-low, |high-prev_close|, |low-prev_close|).
    Первое значение = SMA(TR, period). Дальше Wilder smoothing."""

def sma(values: Sequence[Decimal], period: int) -> Decimal: ...

def ema(values: Sequence[Decimal], period: int) -> Decimal:
    """Стандартный EMA: alpha = 2/(period+1). Первое значение = первое value."""

def percentile_rank(values: Sequence[Decimal], value: Decimal) -> Decimal:
    """Доля values, которые ≤ value. Для ATR-percentile фильтра."""
```

### 3.3 Стратегия `BtcBreakoutStrategy`

State machine:

```
FLAT ──(signal)──▶ PENDING ──(fill)──▶ OPEN ──(TP1 hit)──▶ TP1_HIT
                       │                  │                    │
                       └─(сразу SL)──────▶ CLOSED              └─(SL or trailing)─▶ CLOSED
```

При входе в `OPEN`:
- Адаптер открыл entry + attached SL + attached TP (TP1) — в одном POST.
- Backtester эмулирует то же.

При касании TP1 (`high ≥ tp1_price` для LONG):
- Backtester эмитит `FillEvent(reason=TAKE_PROFIT_1)`. На MVP мы закрываем **всю** позицию по TP1 — упрощение vs план 08 «50%». **TODO в коде** — частичное закрытие — после первой ретро.
- В live эту логику реализуем уже на стороне адаптера: отдельный reduce-only TP-ордер на 50% размера, а оставшиеся 50% — управляются trailing-exit. Тоже отложено.

После TP1 hit → state TP1_HIT (в MVP — сразу CLOSED, потому что мы закрыли всё). State TP1_HIT нужен для будущего trailing.

### 3.4 Алгоритм `on_candle_close`

```
if state == OPEN/TP1_HIT/PENDING → noop (ждём fill / exit)
if state != FLAT → noop

if len(history) < max(donchian_n, atr_window, volume_sma_window) + 1 → noop  # warmup

closed = history[:-1]   # без текущей свечи — Donchian считает по N предыдущим
current = current_candle

# Triggers
long_trigger = current.close > donchian_upper(closed, n=donchian_n)
short_trigger = current.close < donchian_lower(closed, n=donchian_n)
if not (long_trigger or short_trigger): return None

# ATR filter
atr_now = atr(closed, atr_window)
atr_history = [atr(closed[i-atr_window:i], atr_window) for i in lookback_range]
atr_rank = percentile_rank(atr_history, atr_now)
if atr_rank < atr_percentile_min: return None

# Volume filter
vol_sma = sma([c.volume for c in closed[-volume_sma_window:]], volume_sma_window)
if current.volume < vol_sma * volume_multiplier: return None

# Composite
side = "BUY" if long_trigger else "SELL"
funding = funding_provider.get_funding_rate(symbol, current.open_time_ms)
if funding is not None:
    if side == "BUY" and funding > funding_rate_max_pct/100: return None
    if side == "SELL" and funding < -funding_rate_max_pct/100: return None
if news.is_paused(current.open_time_ms): return None
if blacklist.contains(symbol): return None

# Risk
entry = current.close
stop_distance = max(abs(entry - reference_low_or_high), entry * stop_min_pct/100)
stop = entry - stop_distance if side == "BUY" else entry + stop_distance
tp1 = entry + tp1_r * stop_distance if side == "BUY" else entry - tp1_r * stop_distance

inputs = RiskInputs(
    equity=ctx.equity,
    side=Side.LONG if side == "BUY" else Side.SHORT,
    entry_price=entry,
    stop_price=stop,
    ...   # дневной P&L и счётчики — стратегия трекает сама в on_fill
)
decision = risk_engine.evaluate(inputs)
if isinstance(decision, RiskRejection): return None

return OrderRequest(
    symbol=symbol,
    side=side,
    order_type="MARKET",
    quantity=decision.quantity,
    attached_stop_loss=stop,
    attached_take_profit=tp1,
)
```

### 3.5 P&L tracking в стратегии (для RiskEngine)

Стратегия трекает в state:
- `day_pnl` — сумма pnl всех fills с UTC-midnight.
- `day_trades_count` — закрытые сделки сегодня.
- `consecutive_losses` — последние подряд убыточные.

Сбрасывается по UTC-дню в `on_candle_close` если `current.open_time_ms / 86400000 != last_day`.

`week_pnl` / `month_pnl` опциональны — на MVP не считаем, передаём `None` в RiskInputs.
