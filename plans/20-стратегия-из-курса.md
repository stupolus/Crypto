# План 20 — Стратегия из курса «Криптограмотность»

## Дата: 2026-05-15

## Контекст

Бэктест плана 19 (gold/oil/stock + btc_breakout) показал: ни одна
стратегия не имеет доказанного edge на 6 мес истории. btc_breakout
убыточна (PnL -2.4% OOS, PF 0.91), gold — всего 5+1 сделок (статистически
ничего), oil/stock — явно плохо.

Вывод: чистый Donchian-breakout без обоснованных правил входа не работает.
Нужна **проверенная методология**, а не выдуманные параметры.

Пользователь покупал курс «Криптограмотность» — формализуем его правила,
закодируем как стратегию, прогоним на длинной истории по тем же монетам
и другим активам.

## Принцип (CLAUDE.md)

> Любой источник проходит путь: парсим → накапливаем статистику →
> проверяем edge → встраиваем как фильтр или фичу. Не как триггер.

> Сначала план, потом код. Никакого кода до плана.

Курс — это **методология**, а не «чужой сигнал-триггер». Но всё равно:
правила курса → формализация → бэктест → только потом live.

## Фазы

### Phase 0 — Сбор материала (нужен ввод пользователя)
- [ ] Пользователь предоставляет материалы курса (формат — см. ниже)
- [ ] Сырьё кладём в `бизнес/материалы/курс-криптограмотность/raw/`
- [ ] Видео (если есть) → транскрипция Whisper → текст
- [ ] Конспект ключевых правил в `бизнес/материалы/курс-криптограмотность/конспект.md`

### Phase 1 — Формализация правил
- [ ] Извлечь конкретные **проверяемые** правила:
  - Условия входа (что должно совпасть)
  - Условия выхода (TP/SL логика)
  - Тайм-фрейм, инструменты
  - Риск-менеджмент (если описан в курсе)
- [ ] Каждое правило — отдельная testable функция (индикатор/фильтр)
- [ ] Спорные/нечёткие места — список вопросов пользователю

### Phase 2 — Кодификация стратегии
- [ ] `strategies/course_<name>/` — strategy + config + tests
- [ ] Переиспользуем инфру: RiskEngine, BacktestEngine, signals
- [ ] Параметры из курса, НЕ подогнанные под бэктест (anti-overfitting)

### Phase 3 — Бэктест на длинной истории
- [ ] Скачать 2+ года истории (BTC, ETH, SOL + те же gold/oil/stock)
- [ ] Прогон через `scripts.compare_strategies` IS/OOS split
- [ ] Критерий: PF > 1.3 и Sharpe > 0.8 на OOS, ≥ 30 сделок
- [ ] Если не проходит — обратно в Phase 1 (что упустили в правилах)

### Phase 4 — Решение
- [ ] Edge подтверждён → деплой на VST (4+ недели demo)
- [ ] Edge не подтверждён → документируем почему, не торгуем

## 10 причин провала (заранее)

1. Курс описывает дискреционную торговлю (на глаз), не алго-правила →
   формализация теряет суть. Решение: честно выделить только то что
   формализуемо, остальное — как фильтр для человека.
2. Курс рассчитан на ручной вход по «чуйке» графика → нет точных триггеров.
3. Правила курса работали на буле 2020-2021, не на текущем рынке.
   Решение: бэктест на 2+ годах включая медвежку.
4. Overfitting: подгоним параметры «чтобы совпало с курсом-примерами».
   Решение: параметры фиксируем ДО бэктеста, по тексту курса.
5. Look-ahead bias в формализации (правило использует данные будущего).
   Решение: строгая проверка — только закрытые свечи.
6. Курс про спот, мы торгуем перпы с плечом → разная динамика (funding,
   ликвидации). Решение: учесть в риск-модели.
7. Slippage/fees на реальных инструментах съедят edge из курса
   (курс обычно их игнорирует). Решение: backtest с реальными fees.
8. Недостаток данных по RWA/stock perp на BingX (видели в плане 19).
   Решение: бэктест на крипте где история есть, RWA — позже.
9. Психология/дисциплина из курса не переносится в бота (это и плюс).
10. Курс — инфо-продукт, edge мог быть «продать курс», а не торговать.
    Решение: бэктест беспристрастно покажет.

## Формат материалов (что нужно от пользователя)

Любой из:
- **Текст / markdown** — конспект, заметки (вставить прямо в чат или файл)
- **PDF** — методичка курса (Claude читает PDF напрямую)
- **Видео-уроки** — выложить файлы, прогоним Whisper → текст
- **Скриншоты** — слайды/графики из курса (Claude читает изображения)

Чем конкретнее правила (точные условия входа/выхода, цифры), тем
точнее формализация. «Купить на отскоке от поддержки» — недостаточно;
нужно «как определяется поддержка», «что считается отскоком».
