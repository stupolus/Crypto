## Дата: 2026-05-19 · База: планы 31-35, retro forward-демо-старт

## Жёсткое условие

Запущенный forward-демо (config-4h-demo.yaml, config.yaml) —
**pre-registered эксперимент, параметры заморожены**. Менять их «на
ходу» = воид эксперимента + оверфит (главный анти-паттерн сессии).

⇒ Улучшения = **opt-in поля с дефолтами = текущее поведение**. v1
(демо) байт-в-байт прежний. v2 включается только новым
`config-v2.yaml` и проходит ту же валидацию, прежде чем где-то
заменит v1. Улучшения **структурные по теории**, НЕ подгонка порогов
под цифру.

## 3 улучшения (каждое — opt-in, engine-совместимо)

### 43.1 — Confidence-gate
`aggregate_extended_signals` уже считает `confidence_raw` (среднее
уверенностей сработавших детекторов), стратегия его игнорирует.
Добавить `min_confidence: float = 0.0`. Если
`candidate.confidence_raw < min_confidence` → skip. Слабое 2/3 ≠
сильное 3/3. Default 0.0 = поведение не меняется.

### 43.2 — Адаптивный TP R по силе сигнала
Фиксированный `tp1_r_multiple` хрупок. `tp1_r_adaptive: bool = False`;
`tp1_r_min/tp1_r_max` (defaults = tp1_r_multiple). При adaptive:
`R = lerp(tp1_r_min, tp1_r_max, confidence_raw)` — сильнее консенсус →
дальше цель. **Один** attached-TP (engine-совместимо, без правок
core/backtest — partial-TP/trailing требует движка, вне scope).
Default off = без изменений.

### 43.3 — Фильтр режима волатильности (ATR-перцентиль)
У composite ноль рыночного контекста — контртрендовые funding/liq-
входы режет чоп и укатывает тренд. `atr_pct_min: float = 0.0`,
`atr_pct_max: float = 1.0` (= фильтр выключен). Считаем
`percentile_rank` ATR по истории; сигнал только если
`atr_pct_min ≤ rank ≤ atr_pct_max`. Чистый pre-trade gate,
engine-agnostic. Default = пропускает всё.
