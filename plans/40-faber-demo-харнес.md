## Жёсткая рамка (безопасность)

- **НЕ live. НЕ реальные ордера. НЕ торговые ключи.** Харнес
  только: читает публичные данные → считает сигнал →
  СИМУЛИРУЕТ исполнение → пишет лог. Никаких приватных
  BingX-эндпоинтов с правами торговли.
- Стратегия ЗАФИКСИРОВАНА (Faber 200SMA long-or-cash, индекс-
  сигнал). Параметры НЕ трогаем — иначе подгонка.
- Критерий успеха задаётся ДО старта (ниже) и НЕ двигается.
- Live — только отдельным явным «да» владельца ПОСЛЕ ≥4 нед
  demo, прошедшего критерий (CLAUDE.md).

## Предзаданный критерий успеха (фиксируется СЕЙЧАС)

Demo считается пройденным ⇔ ВСЁ выполнено за окно ≥4 недели
(лучше ≥8 для статбазы):
1. **Tracking:** дневная корреляция доходностей перп↔индекс
   ≥0.95 И |медианный базис| ≤0.5% (перп реально следует за
   индексом — иначе сигнал по индексу неприменим к перпу).
2. **Funding-drag:** суммарный уплаченный funding за время
   «в лонге» ≤ 1/3 доходности позиции (иначе премию съедает).
3. **Гэпы:** проскальзывание вход/выход (open релиз-бара vs
   сигнальный close) в пределах смоделированных 0.05–0.10%.
4. **Поведение:** знак и тайминг demo-сделок совпадают с
   тем, что дал бы индекс-бэктест на том же окне (нет
   рассинхрона исполнения).
5. **Риск:** ни одного нарушения лимитов риск-профиль.md /
   circuit breakers за весь demo.
Любой пункт не выполнен → demo НЕ пройден, честно фиксируем,
live не обсуждается.

## Архитектура (40.2)

`scripts/faber_demo_runner.py` (paper, read-only данные):
- daily: тянет индекс close (Yahoo ^NDX или ^GSPC) → Faber
  200SMA сигнал (long / cash);
- тянет BingX-перп close (`NCSINASDAQ1002USD-USDT`, публичный
  quote-эндпоинт, БЕЗ торговых прав) → симулирует позицию/
  funding/гэп;
- риск-сайзинг: фикс % эквити (риск-профиль.md), 1 поз/инстр.;
- пишет `ops/faber_demo.jsonl` (дата, сигнал, индекс, перп,
  базис, позиция, симул-эквити, funding, флаги лимитов);
- идемпотентен (один прогон/день, дозапись), без look-ahead
  (сигнал по ВЧЕРАШНЕМУ закрытию индекса).

## 10 причин провала (априорно, честно)

1. Перп слабо следует за индексом (low corr / большой базис)
   → индекс-сигнал неприменим, demo завален по п.1.
2. Funding на лонг-перпе съедает премию (п.2).
3. Гэпы выходных (индекс закрыт, перп торгуется 24/7) →
   рассинхрон сигнала.
4. ≥4 нед — мало для статвывода; Faber-сделки редки (могут
   не случиться за окно вовсе) → «нет вывода», не «прошёл».
5. Paper ≠ live: латентность/ликвидность не моделируются
   полностью; demo — необходимое, не достаточное.
6. Yahoo индекс-данные с задержкой/ревизией → сигнал шумит.
7. BingX публичный quote может менять символ/режим (24/7 vs
   биржевые часы) — обрабатывать честно.
8. Соблазн «подвинуть критерий» если почти прошёл — ЗАПРЕЩЁН.
9. Один demo-период = один режим; не гарантия будущего.
10. Даже пройденный demo = право на разговор о live, НЕ live.
