## Что считаем «скальпингом» здесь (бэктестируемо)

НЕ стакан/0.1%-тейки/реакция на поток (невалидируемо, HFT-домен).
А: **высокочастотная внутридневная стратегия на 15m-свечах**,
короткое удержание (1–8 баров ≈ 0.25–2 ч), малые цели,
много сделок. Только OHLCV (есть: 15m мажоры).

## Методология (одна предзаданная гипотеза, без подгона)

Каркас «mean-reversion к якорю + фильтр режима»:
- Якорь: VWAP дня ИЛИ EMA(N) на 15m.
- Сигнал: цена ушла от якоря на ≥ k·ATR(15m) → ставка на возврат
  (LONG ниже якоря, SHORT выше).
- Обязательный фильтр: НЕ скальпить в сильном тренде дня
  (|EMA_fast−EMA_slow|/EMA_slow > порог → пропуск; в тренде
  mean-reversion сливает — урок range_reversion).
- Выход: возврат к якорю (TP) ИЛИ k2·ATR стоп ИЛИ N баров.
- Параметры из логики (k≈1.5–2 ATR, окна), НЕ перебор под PnL.
- Риск/сайзинг — текущий RiskEngine (1%/сделку), плечо ≤ как в
  CLAUDE.md. Никакого 25x.

## Критерии валидации (СТРОЖЕ DOLF — обоснование ниже)

| Критерий | Скальп-порог | Почему строже |
|---|---|---|
| Profit Factor (OOS) | **> 1.5** | издержки съедают тонкие движения |
| Sharpe (OOS, ann.) | **> 1.0** | много сделок → нужен явный край |
| Сделок OOS | **≥ 100** | частотная → требуем большую выборку |
| IS≈OOS | да | без режимного флипа (урок funding) |
| Walk-forward | ≥70% окон + | не «одно окно» (урок LTC) |
| Overlap-коррекция | обязательна | коррелированные сделки (урок trailing) |
| **Sweep издержек** | **выжить при 0.10/0.15/0.20% round-trip** | издержки = ГЛАВНЫЙ киллер скальпа |
| Бить B&H по риску | да | иначе это бета/шум |
| Coin-consistency | ≥2/3 мажоров | не cherry-pick (урок l3) |

Скальп проходит ТОЛЬКО при выполнении ВСЕХ. Любой один провал
(особенно cost-sweep) → отклонить, как и всё прочее.
