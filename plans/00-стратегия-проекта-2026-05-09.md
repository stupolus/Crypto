**Дата:** 2026-05-09

## 1. Контекст и цель

Строим **автоматизированную торговую платформу** для крипты, RWA-перпов (золото, серебро, нефть), индексов и токенизированных акций. Реалистичная цель — системно прибыльный бот с разумным риском, не «казино с плечом».

Профиль пользователя:
- Торговал интрадей золото/серебро через BingX (2–3 захода в день, успешно).
- Имел опыт high-leverage / short-stop трейдинга («проба пера»).
- Знает, что можно всё потерять.
- Стартовый депозит: $1 000.
- Целевые фазы: $1k → $10k → $50k → $100k.

## 2. Решённые вопросы (зафиксированы)

### Биржи и инструменты
- **Старт:** BingX (там есть всё под одной крышей: крипта, золото, индексы, акции — пользователь знаком).
- **Источник данных RWA:** Hyperliquid (~52% мирового рынка RWA-перпов).
- **Резервная биржа (фаза 4+):** Bybit, для инструментов которых нет на BingX или для распределения капитала между биржами.
- **НЕ торгуем на:** Hyperliquid (риск вмешательства биржи как в JELLY-инциденте), Polymarket (другая природа эджа).

### Стратегия №1
- **Тип:** Momentum / Breakout (классика — пробой N-периодного максимума с подтверждением).
- **Почему:** простые правила, бэктестируется, совпадает с природой крипты (fat tails трендов), стопы структурные.
- **Инструмент фазы 1:** BTC-USDT perp на BingX.
- **Расширение:** позже добавляем mean reversion (после ликвидаций) и volatility breakout как отдельные стратегии (фаза 3+).

### Риск-каркас (см. [[бизнес/риск-профиль]])
- 1.0% риска на B-tier сделку, 1.5% на A, 2.0% на A+.
- Дневной стоп −3%, недельный −7%, месячный −15%.
- Макс. эффективное плечо 5x. Биржевое плечо ≤ 5x.
- 1 позиция за раз на инструмент.
- Изолированная маржа всегда. Кросс — никогда.
- Стоп ставится одновременно с входом.

### Анти-скальпинг
- Никогда плечо > 5x.
- Никогда стоп < 0.5%.
- Никогда цель TP < 0.5%.
- Сильные сетапы используем **размером**, не плечом.

## 4. Фазы (дорожная карта)

### Фаза 0: Платформа (4–6 недель)

3. Risk engine: формула размера позиции, циркуит-брейкеры, blacklist.
4. Парсер TG-скринеров (read-only, копит данные с дня запуска).
5. Парсер Hyperliquid (ликвидации, OI, funding).
6. Backtest engine: на исторических свечах + collected screener events.
7. Telegram-алерты (kill switch, exit-сделка, аномалия).
8. ✅ Аудит инструментов BingX → заполнение `бизнес/инструменты-bingx.md` (выполнено в фазе 0.A, см. [[plans/01-bingx-адаптер]] §13). ### Фаза 1: BTC breakout (6–10 недель: 4 testnet + 4 live)

**Стратегия:** Donchian breakout с подтверждением (ATR + volume + composite state).

**Сетап (предварительно, уточняется в `plans/02-btc-breakout.md`):**
- Long-вход: цена пробивает максимум N=20 свечей на 15m TF + объём > 1.5× SMA(20) + ATR-фильтр волатильности.
- Стоп: за минимум диапазона N свечей, минимум 0.5% от цены.
- TP1 (50% позиции): 1.5R или перед ближайшим кластером ликвидаций (что ближе).
- TP2 (50% позиции): trailing stop по EMA21.
- Композитные фильтры:
  - не вход, если тикер в blacklist (manipulation pattern).
  - не вход, если funding > 0.05% за 8ч (extreme positioning против нас).
  - не вход за 30 мин до и после крупной новости (Forex Factory high-impact).

**Чек-лист:**
- [ ] Стратегия в коде.
- [ ] Бэктест на BTC-USDT 6+ месяцев → результаты в `plans/02-btc-breakout-backtest.md`.
- [ ] 4+ недели на testnet → expected statistics.
- [ ] **Подтверждение пользователя на live** (ручное, явное).
- [ ] $1 000 на live, 4–8 недель.
- [ ] Drawdown < 15% за период.
- [ ] Sharpe > 1.0 за период.

## 5. 10 причин провала (критика)

Прежде чем начать кодить — продумываем, что может убить проект.

1. **BingX API нестабилен / меняется без предупреждения.**
   - Митигация: версионируем адаптер, тесты на каждое обновление, fallback на REST если WS падает.

2. **Стратегия overfit на бэктесте, на live не работает.**
   - Митигация: разделение train/test, walk-forward analysis, бэктест на out-of-sample данных, медленный rollout (testnet → мини-капитал → масштаб).

3. **TG-скринеры — шум, ничего не предсказывают.**
   - Митигация: 4 недели сбора данных + статистика **до** использования в торговле. Если edge нет — выкидываем.

4. **Crypto crash / манипуляция → стопы выносит.**
   - Митигация: дневной/недельный стоп, изолированная маржа, blacklist манипулируемых пар.

5. **API-ключи компрометированы.**
   - Митигация: ключи только торговые (без вывода), IP whitelist, мониторинг логов, rotation 1× в 3 месяца.

6. **VPS падает / сетевые проблемы.**
   - Митигация: kill switch при разрыве WS > 30 сек, redundancy на двух VPS (фаза 3+), мониторинг latency.

7. **Жадность / страх → ручное вмешательство в бот.**
   - Митигация: правило «не трогать бот вживую»; все изменения через план + бэктест. Анти-скальпинг файл как напоминание.

8. **Регуляторика по тебе/бирже меняется.**
   - Митигация: распределение по биржам, юрисдикциям; не вкладываем всё в один venue.

9. **Funding rate накапливается, съедает прибыль.**
   - Митигация: учитываем в стратегии. Не держим позиции > 8ч если funding негативный для нас.

10. **«Слишком хорошо работает» → масштабируем слишком быстро → один drawdown больно.**
    - Митигация: жёсткие фазы, явные подтверждения переходов, минимальные периоды на каждой фазе.

## 9. Резюме

**Что было:** разрозненные идеи в чате (BingX, Bybit, скринеры, Polymarket, Hyperliquid, скальпинг, ML, новости, киты).

**Что стало:** единый мастер-план с фазами, чёткими условиями переходов, рисковым каркасом, анти-паттернами и backlog'ом. Все идеи распределены: что в фазу 1, что в фазу 2+, что в backlog, что в категорию «не делаем никогда».

**Какую проблему решал:** перевод хаоса в дисциплинированный roadmap. Защита депозита от типичных ритейл-ошибок (скальпинг с плечом, копирование сигналов, Франкенштейн-стратегия).

**Было ли это самым эффективным решением:** да. Дисциплина методологии Смыслокода (план до кода, второй мозг, фазовая разработка) — единственный надёжный путь для проекта такого масштаба.
