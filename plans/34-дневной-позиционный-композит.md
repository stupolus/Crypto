# План 34 — Дневной позиционный композит (crowd + funding)

## Контекст и решение

Скальп закрыт доказательно (план 33.1–33.11: price-pattern нет
edge, taker+maker, 45 корзин). Реальный, робастный edge найден
в **позиционировании** (33.13): контр-крауд по account L/S ratio —
PF 1.2–1.5, walk-forward 3–4/4, ~1200д, 6 мажоров. Сам по себе
модест (t≈1, Sharpe~0.7). Решение владельца: развить (план 34).

Гипотеза: **диверсификация слабых-но-реальных краёв** (как
работают фонды) поднимает значимость честно. Композит из двух
предзаданных, тезис-обоснованных сигналов:
- S1 крауд-контр: global account L/S ratio в крайних 20% → фейд.
- S2 funding-экстремум-контр + liq-подтверждение (33.12, +EV
  на сплите, WF-слаб сам по себе).
Вход ТОЛЬКО при консенсусе S1∧S2 (одно направление) — выше
конвикция, и если края частично независимы → t растёт честно.

**Это НЕ скальп.** Дневной позиционный сигнал — то, что данные
реально позволяют. Ожидание скромное; «прошёл» = право на demo,
НЕ деньги.

## Правила (предзадано, НЕ скан под PnL)

- Экстремум = крайние 20% метрики самой монеты (как 33.12/13).
- Горизонт удержания: 1 день (предзадано; 3д — вторичная
  проверка робастности, не отбор лучшего).
- Консенсус: позиция если S1 и S2 дают одно направление; иначе
  flat. Сайзинг — фикс риск % (риск-профиль.md), равный вес
  6 мажоров; 1 позиция/инструмент (CLAUDE.md).
- Look-ahead исключён: метрики и SMA строго из прошлых баров.

## Гейт (СТРОГИЙ, НЕ ослабляется)

OOS: PF>1.3 ∧ Sharpe>0.8 ∧ **t>2** ∧ ≥8 нед.корзин;
walk-forward ≥3/4 фолдов + ; cost-sweep 0.04/0.07/0.10% —
все уровни; multiple-testing учтён (≤4 предзаданных вида,
нужен явный устойчивый +, не пограничный). Любой провал
cost/WF → отклонить.

## 10 причин провала (априорно, честно)

1. S1 и S2 коррелированы (оба контр-позиционные) → нет
   диверсификации → t не растёт.
2. Консенсус режет выборку → <8 корзин → нет статбазы.
3. S2 сам WF-слаб (33.12 1/4) → тянет композит вниз.
4. Контр-сигнал = риск катастрофы в сильном тренде (фейд
   ралли) — хвостовой риск, Sharpe обманчив.
5. ~1200д = 1 макроцикл; OOS всё равно один режим.
6. Multiple-testing по всей сессии (≥40 тестов) → даже t>2
   тут может быть ложным; нужен demo как финальный арбитр.
7. Account-ratio данные Binance, торгуем BingX — прокси-риск.
8. Дневной сигнал = мало сделок/год → медленный набор доверия.
9. Costs на дневном меньше, но проскальзывание на входе по
   сигналу close→next-open реально.
10. Даже идеальный бэктест ≠ деньги: только demo решает.
