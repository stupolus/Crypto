# План 35 — composite: 3-й сигнал из CVD (кросс-источник)

## Дата: 2026-05-18 · База: план 31, retro гейт-блокирован

## Находка (диагностика, не провал)

composite 4h на чистом окне (~5 мес, все свечи с данными):
**2 сделки за 5 мес** → невалидируемо. Корень — НЕ рынок, а
конструкция: `aggregate_extended_signals` хочет 2-of-3, но
`order_flow_signal=None` (нет провайдера bid/ask) → работают 2
детектора из 3 → «2-of-3» вырождается в «оба редких сигнала
(funding_extreme И liquidation_sweep) совпали по направлению» = почти
никогда.

Ответ ровно по запросу пользователя («кросс получение информации»):
**CVD уже бэктестится** (`backfill_providers` отдаёт `_delta`), но
стратегия его игнорирует. Подключаем CVD как 3-й голос → 2-of-3
работает как задумано (любые 2 из 3), частота сделок вырастает до
валидируемой.

## Что меняем (money-path — поэтому план первым)

### 35.1 — composite_signal: 3-й сигнал из CVD
- `CompositeSignalStrategy.__init__` += `delta_provider: DeltaProvider`.
- Каждую свечу: `cvd = delta.get_cvd_series(symbol, ts, cvd_lookback)`;
  per-bar приращения → `buy = Σ(+Δ)`, `sell = Σ|−Δ|`;
  `detect_order_flow(buy, sell, OrderFlowConfig(threshold))` →
  `OrderFlowSignal`. Передаём 3-м аргументом в
  `aggregate_extended_signals` (переиспользуем оттестированный
  детектор, без новой сигнальной логики).
- config += `cvd_lookback:int`, `order_flow_threshold:float`
  (дефолт детектора 0.6 — pre-registered, не подгоняем).

### 35.2 — wiring
`composite_backtest`/`run_backtest`/`live_runner`: прокинуть
`delta_provider` (в backtest — из `backfill_providers` `_delta`).

### 35.3 — тесты
Новый кейс: 3 голоса, любые 2 совпали → ордер; CVD один не даёт
консенсуса. Существующие 6 — зелёные.

### 35.4 — ре-валидация
`composite_backtest` BTC/ETH 4h в Coinglass-окне. Смотрим: число
сделок стало валидируемым? Вердикт по iter#4 честно, параметры НЕ
трогаем.

## Ограничения (честно)
- Coinglass-тариф: только 4h+, история ~с 2025-11 (~6 мес), 429.
  Даже с 3-м сигналом полноценный 12-мес/9-окон WF невозможен — будет
  «сильный сигнал / нет», не «доказано». Это шаг к валидации, не сама
  валидация.
- Другие биржи/интервалы НЕ решают (история тарифа — бинд). Реальная
  валидация всё равно требует апгрейда тарифа ИЛИ forward-демо.
