# План 46 — Macro lead-lag: ищем настоящий info-edge (вариант B)

## Дата: 2026-05-19 · База: разговор «game changer ≠ ещё крутилка»

## Зачем

Edge через ещё-одно-улучшение strategy.py на 4-7 сделках = подгонка.
**Game-changer возможен только через новый info-edge.** Принцип 3:
композит важнее сырого; добавить НОВЫЙ ортогональный источник.

Кандидат: **macro-факторы** (DXY, VIX → BTC). Свободные данные с
длинной историей (Yahoo, без ключа), известная гипотеза (DXY/VIX как
risk-off → крипта реагирует). Если статистически докажем
**опережение** — это 4-й голос в composite. Если нет — закрываем
гипотезу честно.

## Pre-registered гипотезы (до прогона данных)

H0 (null): дневное изменение DXY (или VIX) не опережает дневное
изменение BTC на лагах 1..3 дня.

H1: при значимом росте VIX (или DXY) на день T BTC доходность на
день T+1 систематически смещена (контртрендовая контра-DXY/контра-VIX).

Тесты (оба):
- **Lag-correlation**: corr(ΔDXY_t, ΔBTC_{t+k}) для k=1,2,3; Pearson +
  permutation p-value (1000 шаффлов), α=0.05.
- **Conditional mean**: top/bottom децильные дни по ΔDXY → среднее
  ΔBTC_{t+1}; t-тест на разницу, α=0.05.

Корректировка множественности (Bonferroni по числу тестов).

## DoD первого шага (этот PR)

- `scripts/macro_edge_probe.py`: тянет BTC-USD, ^VIX, DX-Y.NYB
  дневные с Yahoo (повторно `download_equity`), считает оба теста,
  печатает p-values + эффект-размеры.
- Тесты pure-функций (без сети): расчёт lag-corr и conditional-mean.
- Прогон на реальных данных, **честный** вердикт в retro:
  - p < 0.05 после Bonferroni → план интеграции 4-го голоса (отд. PR).
  - p ≥ 0.05 → гипотеза отклонена, переходим к следующему кандидату
    (twitter sentiment / on-chain) или закрываем направление.

## Жёстко (анти-фишинг)

- Тесты фиксируются ДО смотрения на p-value. Не «прикручиваем» лаги/
  децили задним числом.
- Если p ≈ 0.05 на одном тесте из двух — НЕ значимо (правило: ОБА
  ниже Bonferroni-порога).
- Не трогаем composite/strategy.py пока H0 не отклонена.

## Не делаем

- Twitter/on-chain пока — отдельные планы при отрицательном macro.
- Никаких «давайте попробуем 7 лагов» — только pre-registered 1..3.
