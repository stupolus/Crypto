# План 39 — EIA crude inventory: событийная спекуляция (Принцип 6)

## Контекст

Запрос владельца: спекуляция на нефти «по новостям». По
Принципу 6 (CLAUDE.md) — НЕ дискреционный news-trigger, а
квантифицированная событийная фича → строгая edge-проверка
→ правило. EIA Weekly Petroleum Status Report (crude oil
inventories) — публичный, регламентный, **квантифицируемый**
релиз (среда 10:30 ET, ~еженедельно с 1990-х). Это легитимный
кандидат: не «новость вообще», а измеримый сюрприз = факт −
консенсус (или факт − прогноз-прокси).

Априор честный: систематический ценовой шорт нефти по тренду
убыточен (план 37.5). Событийный сигнал — ДРУГОЙ (поток
информации в момент релиза), прежний вердикт его не выносит,
но и не обещает плюс. Проверяем строго, ждём, скорее всего,
слабый/нулевой edge после издержек.

## Данные (честно про доступность — ключевой риск)

Нужно:
- EIA crude inventory: фактический недельный change (EIA API
  v2 `petroleum/stoc/wstk`, открытый; reachability из песочницы
  — ПРОВЕРИТЬ в 39.2, не предполагать).
- Консенсус/прогноз: труднодоступен бесплатно. Прокси сюрприза
  (предзадано, не подгон): фактический change − его сезонная
  норма (тот же ISO-week среднее за прошлые годы, расширяющееся
  окно, без look-ahead) ИЛИ vs 4-нед скользящее ожидание.
- Цена: WTI 1d (Yahoo CL=F, есть) для forward-доходности на
  релизный день и +1/+3 дня.
Если EIA API недостижим из песочницы → план честно
останавливается на «нет данных», НЕ выдумываем (как с
интрадей-DOLF/перп-историей).

## Правило (предзадано, НЕ скан)

- Событие: день публикации EIA crude change.
- Сюрприз s = change − ожидание (сезонная норма / 4-нед MA).
- Логика (документированная): неожиданный РОСТ запасов =
  медвежий для нефти → SHORT WTI на закрытии релиз-дня;
  неожиданное ПАДЕНИЕ = LONG. Удержание 1 и 3 дня (предзадано,
  не отбор лучшего). Экстремум сюрприза = крайние 30%
  собственного распределения (предзадано).
- Без плеча сверх риск-профиль.md; кост round-trip 0.10%.

## Гейт (СТРОГИЙ, не ослабляется)

OOS (split по медиане даты): PF>1.3 ∧ Sharpe>0.8 ∧ t>2 ∧
≥30 событий-в-OOS ∧ ≥8 «корзин» (квартал/месяц); walk-forward
≥3/4; cost-sweep 0.07/0.10/0.15%. Multiple-testing: ≤4
предзаданных среза (2 горизонта × {сезонная норма, 4-нед MA}),
нужен явный устойчивый +, не пограничный. Любой провал WF/cost
→ отклонить честно.

## Фазы

- 39.1 (этот файл) план.
- 39.2 `scripts/eia_probe.py`: проверка достижимости EIA API
  из песочницы + забор истории crude change. Нет доступа →

- 39.3 `scripts/eia_oil_event_eval.py`: предзаданное правило,

- 39.4 Вердикт. Прошёл строгий → отдельный план demo (не
  деплой). Не прошёл → честно закрыть, это валидный итог.

## 10 причин провала (априорно, честно)

1. EIA API недостижим из песочницы → нет данных, стоп.
2. Сюрприз без реального консенсуса = шумный прокси.
3. Реакция нефти на EIA уже в цене за секунды (эффективность)
   → дневной вход опоздал.
4. WTI CL=F = front-future с roll/контанго, не спот.
5. Малая выборка релевантных «экстремальных» событий → нет
   статбазы / нет ≥8 корзин.
6. Look-ahead в «ожидании» если не расширяющееся окно — строго
   избегаем.
7. EIA-эффект нестабилен по режимам (сланец/COVID/война).
8. Издержки + проскальзывание в момент релиза выше обычного.
9. Multiple-testing сессии → нужен demo как арбитр.
10. Даже + бэктест ≠ деньги: решает demo (план как 38→39).

## Жёсткие стопы

- НЕ news-trigger, НЕ дискреция: только квантиф. сюрприз →
  предзаданное правило → edge-гейт (Принцип 6 / Принцип 1).
- Параметры из логики, не подгон под Sharpe.
- Нет данных EIA → честный «стоп», не выдумка.
- Не live/не demo здесь. cost-sweep/WF/t>2 не ослабляются.
- «Прошёл» = право на demo, НЕ деньги.

## 39.2 ВЕРДИКТ (2026-05-17) — данные EIA НЕДОСТИЖИМЫ (честный стоп)

Проверено из песочницы:
- EIA API v2 (все эндпоинты, без ключа): **HTTP 403 Forbidden**
  — требует бесплатный api_key (регистрация eia.gov).
- EIA v2 seriesid / legacy: тоже 403.
- FRED CSV (WCESTUS1/WCRSTUS1, та же серия запасов): **HTTP 503**
  — недостижим из песочницы (как Stooq ранее в сессии).

Следствие: квантифицированный сюрприз запасов нефти получить
нельзя → событийное исследование 39.3 **не запускается**.
Честный стоп по data-wall (как интрадей-DOLF, перп-история) —
НЕ выдумываем результат (план это предзадал).

**Разблокировка (действие владельца):** бесплатный EIA API key
(eia.gov/opendata) → в `.env` как `EIA_API_KEY` (по аналогии
с `COINGLASS_API_KEY`). После этого 39.3 (предзаданное правило
+ строгий гейт) выполнимо. Без ключа план корректно
приостановлен, не закрыт отрицательно (данных нет ≠ edge нет).

## 39.2/39.3 ВЫПОЛНЕНО (2026-05-19) — ключ получен

Владелец прислал `EIA_API_KEY`. Probe пофикшен (v2-фасет
`WCESTUS1`, не legacy `PET.WCESTUS1.W`): забрано **2276 недель**
коммерческих запасов нефти (1982→2026) → `data/eia/`

6542 дн. бара 2000→2026 (пересечение с EIA ≈ окно событий).

## 39.4 ВЕРДИКТ (2026-05-19) — НЕ ПРОШЁЛ строгий гейт

`scripts/eia_oil_event_eval.py`, 4 предзаданных среза
(2 горизонта × 2 модели ожидания), OOS split по медиане даты,
WF 4 фолда, cost-sweep 0.07/0.10/0.15%:

| Срез | OOS PF | Sharpe | t | WF | Гейт |
|---|---|---|---|---|---|
| seasonal h1 | 0.70 | −0.10 | −1.50 | 1/4 | ❌ |
| seasonal h3 | 1.06 | 0.02 | 0.25 | 3/4 | ❌ |
| ma4 h1 | 1.02 | 0.01 | 0.12 | 1/4 | ❌ |
| ma4 h3 | 1.19 | 0.05 | 0.90 | 3/4 | ❌ |

Ни один срез не близок к гейту (нужно PF>1.3 ∧ Sharpe>0.8 ∧
t>2 ∧ ≥30 OOS ∧ ≥8 корзин ∧ WF≥3/4). Лучший (ma4 h3):
PF 1.19, t 0.90 — статистически неотличимо от нуля; на
издержках 0.15% PF падает к 1.10. seasonal h1 даже
отрицателен. Совпадает с честным априором плана (п.3,7 «10
причин»): реакция нефти на EIA в цене за секунды, дневной
вход опаздывает; прокси-сюрприз без реального консенсуса шумный.

**Итог честный и валидный (план это предзадал):** событийная
EIA-нефть как систематическое правило edge НЕ даёт. Закрываем
отрицательно. Принцип 6 соблюдён: спекулятивную идею провели
через конвейер (данные → строгая проверка → правило), а не
в дискреционный live-триггер. Артефакт `eia_oil_event_eval.py`
оставлен как проверяемый, в торговлю не идёт. План 39 ЗАКРЫТ.
