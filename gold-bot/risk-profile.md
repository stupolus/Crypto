## Декларация капитала

- **Полный декларированный банк проекта:** $100 000.
- **Активный торговый банк (в работе у бота):** определяется текущей фазой (см. ниже).
- **Резерв вне биржи:** всегда ≥ 80% полного банка, пока проект не пройдёт фазу 3.

Декларированный банк = размер, под который проектируется риск-модель. Активный банк = что реально на бирже.

## Фазы капитала

Проект движется по фазам. Переход между фазами — **только по явному «да» пользователя**, не автоматически.

| Фаза | Режим | Активный банк | Условия перехода вперёд |
|---|---|---|---|
| **0. Setup** | бэктест + paper | $0 на бирже | Инфраструктура готова, ≥ 1 стратегия с OOS-валидацией |
| **1. Paper / testnet** | Bybit testnet + BingX paper | $0 реальных | 4+ недели без критических багов, PF ≥ 1.3 на OOS-окне, max DD ≤ 8% |
| **2. Mini-real** | live | $1 000 | 4+ недели live, PF ≥ 1.2, max DD ≤ 6% от $1k, нет нарушений правил |
| **3. Pilot** | live | $5 000 | 6+ недель, метрики не хуже фазы 2 пересчитанные, нет инцидентов |
| **4. Scale** | live | до 20% полного банка ($20k) | По отдельному решению, с обновлением риск-профиля |
| **5. Full** | live | по решению | Никаких автоматических «полных $100k». Если когда — отдельным планом. |

Фаза по умолчанию: **0**. Переход в фазу 1 разрешается планом инфраструктуры. Переход в фазу 2+ — каждый раз отдельным разрешением.

## Риск на сделку

| Параметр | Значение | Почему именно так |
|---|---|---|
| Базовый риск на сделку | **0.5% активного банка** | Консервативно для проекта на старте. Поднимем после фазы 3 если статистика устойчивая. |
| Максимум на сделку (даже A+ сетап) | **1.0% активного банка** | Жёсткий потолок. A+ сетап не повод удваивать риск дважды. |
| Минимум на сделку | **0.2% активного банка** | Ниже — шум не отбивает комиссии. |

## Лимиты просадки (circuit breakers)

| Триггер | Действие |
|---|---|
| Дневной PnL ≤ −2% активного банка | STOP всей торговли до следующего календарного дня (UTC) |
| 4 убытка подряд | STOP всей торговли до следующего календарного дня |
| Недельный PnL ≤ −5% активного банка | STOP до понедельника, обязательное ретро в `journal/` |
| Месячный PnL ≤ −10% активного банка | STOP проекта, ручной разбор и решение о продолжении |
| **Глобальная просадка от пика эквити ≤ −15%** | **KILL-SWITCH: полная остановка бота + закрытие позиций + критический алерт пользователю. Перезапуск только вручную после разбора.** |
| WS-disconnect > 30 сек | Все открытые позиции закрываются рыночным, торговля на паузе до восстановления |
| Ack ордера > 10 сек | Алерт, не закрываем, флагуем |
| Волатильность инструмента > 3% за 5 мин | Пауза по этому инструменту на 60 мин (новость/паника) |
| Spread > 3x от 30-дневной медианы | Не торгуем этот инструмент до нормализации |

## Плечо

| Параметр | Значение |
|---|---|
| Максимум эффективного плеча | **5x** (notional / equity по конкретной позиции) |
| Биржевой ползунок плеча | Может быть выше — это запас маржи, не цель. Жёстко проверяем эффективное плечо перед каждым ордером. |
| Маржинальный режим | Только **isolated**. Cross запрещён инвариантом. |

## Размер позиции

```
risk_per_trade   = active_capital * risk_pct
stop_distance    = |entry - stop| / entry
notional         = risk_per_trade / stop_distance
position_size    = notional / entry
effective_lev    = notional / active_capital
```

Перед каждым ордером проверяется:
- `effective_lev <= 5` — иначе ордер не отправляется.
- `stop_distance >= 0.2%` — стопы тоньше = шум.
- Стоп ставится одновременно с входом (attached). Если биржа не подтвердила стоп в течение N мс — позиция закрывается рыночным.
- Расстояние от стопа до расчётной цены ликвидации ≥ 30% от расстояния входа до стопа.

## Одновременные позиции

| Параметр | Значение |
|---|---|
| Макс. одновременных позиций | **3** |
| Макс. позиций в одном классе (металлы / equity / crypto) | **2** |

## Частота сделок

| Параметр | Значение |
|---|---|
| Макс. сделок в день суммарно | **15** |
| Макс. сделок по одному инструменту в день | **5** |
| Минимальный интервал между двумя сделками по одному инструменту | **5 минут** |

Частота — не цель. Это потолок защиты от багов / overtrading.

## Окна торговли

| Класс | Окно (UTC) | Почему |
|---|---|---|
| Equity-перпы (TSLA/NVDA/AAPL/META) | **13:30–20:00 UTC** | Только основная сессия NYSE. Pre-/post-market на токенизированных перпах = тонкая ликвидность. |
| Металлы (PAXG/XAUT) | 24/7 разрешено, но **боевые окна**: 07:00–22:00 UTC | Лондон + NY перекрытие — основная ликвидность по золоту. |
| Crypto (BTC/ETH/SOL — калибровка) | 24/7 | Калибровочный набор, оценка работоспособности инфраструктуры. |

За 30 мин до и 15 мин после high-impact событий (FOMC, CPI, NFP, ECB, earnings по торгуемому тикеру) — **новые входы запрещены** по затрагиваемому классу.

## Стоимостная модель (для бэктеста и предторговой проверки)

Используется и в бэктесте, и в RiskEngine перед отправкой ордера.

- BingX perp taker fee: из конфига (актуальное значение с источником, не из памяти).
- Bybit perp taker fee: из конфига.
- Maker fee: из конфига.
- Funding rate: исторический по конкретному инструменту из конфига/кэша.
- Slippage: tier-based на наблюдаемом спреде × множитель размера. Множитель — в конфиге.

Правило **cost-edge gate**: ожидаемый TP должен покрывать суммарную стоимость сделки минимум в **2 раза**, иначе сделка отклоняется ещё на этапе сигнала.
