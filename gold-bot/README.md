# gold-bot

**Что это.** Отдельный проект внутри репозитория. Автоматизированный интрадей-бот на токенизированном золоте/серебре (PAXG, XAUT) и токенизированных equity-перпах (TSLA, NVDA, AAPL, META), плюс крипто-мейджоры (BTC, ETH, SOL) как калибровочный набор. Биржи: BingX и Bybit. Банк проекта декларируется в `risk-profile.md`, но **в реальную торговлю ничего не идёт без явного письменного «да» по каждой стратегии**.

**Зачем отдельная папка.** В корне репозитория уже живёт другой, развитый крипто-проект со своими правилами (`/CLAUDE.md`, `/AGENTS.md`, `/бизнес/`). Этот проект — новый, с собственными правилами, чтобы случайно не подмешать старые предположения. Когда работа идёт **внутри `gold-bot/`** — действуют правила из `gold-bot/CLAUDE.md`. Когда в любой другой папке репозитория — правила корня.

**Статус.** Этап 0: правила, без кода. Никакой стратегии, адаптера, бэктестера пока нет. Любой код — только после плана (см. `gold-bot/CLAUDE.md` → раздел «Поток работы»).

**Файлы.**
- `gold-bot/CLAUDE.md` — главные правила. Читается каждой сессией, работающей в этой папке.
- `gold-bot/risk-profile.md` — все числовые риск-параметры. Единственный источник истины для цифр.
- `gold-bot/plans/` — техпланы по фичам. Один файл = одна фича. Кода до плана нет.
- `gold-bot/journal/` — журнал сделок и ежедневные/еженедельные ретро. Появится с первым paper-запуском.

**Принцип честности (главное).** Бот не «приносит пассивный доход». Бот — это система с положительным матожиданием, которую мы строим, проверяем и наблюдаем. Никаких гарантий, никаких обещаний доходности. Любая стратегия проходит paper → testnet → mini-real → масштаб. Между этапами — недели наблюдения, не дни.
