# 04 — Event-driven бэктестер + walk-forward

Дата: 2026-05-22
Статус: в работе (goal Шаг 2). Реализуется в той же сессии.

---

## Цель

Честный event-driven бэктестер без lookahead, с реальными издержками (комиссии +
слиппедж + funding), метрики и walk-forward (IS/OOS) — чтобы отбраковывать
стратегии до любых реальных денег.

## Гипотеза edge'а

Edge'а нет — это инструмент проверки edge'а. Главное — не врать себе: нет lookahead,
издержки в каждой сделке, OOS обязателен.

## Что входит

### 4A — ядро
- `backtest/strategy.py`: Protocol `Strategy` (`on_candle(history) -> Signal | None`),
  `Signal` (side, stop, take_profit, risk_pct, asset_class).
- `backtest/costs.py`: `CostModel` (taker/maker/slippage из конфига) + расчёт издержек входа/выхода.
- `backtest/engine.py`: `BacktestEngine` — одна позиция за раз (v1), MARKET-fill по
  open(c+1), стоп/тейк внутри бара по high/low (консервативно: при касании обоих в
  одном баре считаем срабатывание стопа), издержки на вход и выход, журнал сделок +
  кривая эквити. Стратегия видит `history[:i+1]`, не видит будущее.
- `backtest/metrics.py`: PF, expectancy, winrate, max DD, Sharpe, Sortino, ulcer,
  num_trades, avg_trade_after_costs.
- Тесты, в т.ч. **lookahead-independence**: изменение будущих свечей не меняет
  прошлые сигналы/сделки.

### 4B — walk-forward
- `backtest/walkforward.py`: rolling IS/OOS окна (split-fraction, шаг), агрегация
  OOS-метрик. Возвращает per-window метрики + суммарную OOS.

## Данные / источники

- `marketdata.load_parquet` (plan 02) — свечи.
- `risk.RiskConfig`/`compute_sizing` (plan 03) — размер позиции.
- Издержки — из cost-конфига (taker/maker/slippage), значения с источником.

## Метод проверки

- Unit: известный сценарий (1 сделка, известный PnL после издержек).
- Unit: стоп срабатывает по low (long) / high (short); тейк по противоположному.
- Unit: оба уровня в одном баре → считаем стоп (консервативно).
- Unit: lookahead-independence (ключевой).
- Unit: метрики на известном ряде сделок (PF, DD, expectancy).
- Unit: walk-forward делит и агрегирует корректно.

## 10 причин почему может не сработать

1. Lookahead через fill по close(c) вместо open(c+1) — фиксируем филл по следующему открытию.
2. Стоп и тейк в одном баре — нельзя знать порядок внутри бара → консервативно стоп.
3. Издержки забыты на выходе — считаем на обе ноги.
4. Подгон под всю историю — walk-forward с OOS обязателен.
5. Незакрытая последняя свеча — бэктест не торгует по последнему open (нет c+1).
6. Decimal-деление → хвосты; контролируем, сравнения через допуск в тестах где нужно.
7. Survivorship/гэпы в данных — бэктест работает по факту, не заполняет.
8. Sharpe завышается на низкой частоте — смотрим PF/expectancy/DD вместе.
9. Слиппедж-модель слишком оптимистична — берём консервативный tier, сверяем с paper.
10. Одна позиция за раз — упрощение v1; портфельный бэктест — отдельный план.

## Критерии успеха

- lookahead-independence тест зелёный.
- Известный сценарий даёт ожидаемый PnL после издержек.
- Метрики считаются корректно.
- walk-forward даёт per-window + OOS агрегат.
- ruff + format + mypy --strict чисто, pytest зелёный.
