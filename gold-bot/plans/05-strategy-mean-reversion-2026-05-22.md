# 05 — Стратегия №1: интрадей mean-reversion золота/серебра у VWAP±ATR

## Цель

Первая проверяемая стратегия: интрадей возврат к VWAP на токенизированном золоте/
серебре (и крипто-калибровка), проверить через walk-forward с реальными издержками.

## Гипотеза edge'а

Внутридневная цена металлов склонна возвращаться к объёмному VWAP после резкого
отклонения (ликвидность вокруг VWAP); отклонение нормируем на ATR (режим
волатильности).

## Правила (точные)

- Таймфрейм: 15m (интрадей, не HFT).
- VWAP по окну `vwap_window` свечей; ATR по `atr_period`.
- **Long:** close < VWAP − `k_entry`·ATR. Stop = close − `k_stop`·ATR. TP = VWAP.
- **Short:** close > VWAP + `k_entry`·ATR. Stop = close + `k_stop`·ATR. TP = VWAP.
- Сессия: только в боевом окне инструмента (золото 07:00–22:00 UTC — risk-profile).
- Размер — через RiskEngine (риск `risk_pct_base`). Стоп обязателен.
- Параметры (vwap_window, atr_period, k_entry, k_stop, session) — в
  `strategies/mean_reversion_vwap/config.yaml`, не хардкод.

## Данные / источники

- Исторические 15m-свечи (plan 02, `load_parquet`) — **скачиваются на VPS** (из
  контейнера сети нет). Бэктестер (plan 04), RiskEngine (plan 03), индикаторы.

## Метод проверки

- Unit (без сети): на синтетике — отклонение ниже полосы → long с правильными
  stop/tp; выше → short; внутри полосы → None; вне сессии → None; мало данных → None. ✅
- **OOS-вердикт по реальным данным — на VPS** через `scripts/run_backtest.py`
  (walk-forward, ≥3 OOS-окна, ≥2 символа). В контейнере — только логика.

## 10 причин почему может не сработать

1. Mean-reversion ломается в тренде — отклонение продолжается, стоп. Нужен тренд-фильтр?
2. На тонком золоте VWAP шумный — ложные сигналы.
3. Издержки съедают малый возврат к VWAP (TP близко) — cost-edge gate отбракует.
4. k_entry/k_stop подогнаны под историю — фиксируем логикой (1.5–2·ATR), OOS решает.
5. Сессионный VWAP должен сбрасываться по сессии; rolling-окно — упрощение v1.
6. Гэпы/паузы металла ломают ATR — сессионное окно снижает.
7. Одна позиция за раз (бэктест v1) — пропускает параллельные сигналы.
8. Мало сделок (<100) → незначимо — отбраковка.
9. Отсутствие золота-перпа в ЕЭЗ → тестируем на крипте-калибровке, ждём recon.
10. Переобучение на одном режиме рынка — walk-forward по разным периодам.

## Критерии успеха (OOS, на реальных данных)

По master-плану: PF ≥ 1.3, expectancy > 2× cost, max DD ≤ 8%, ≥ 30 сделок/OOS-окно,
≥3 OOS-окна на ≥2 символах. Иначе — честно отбраковываем (фиксируем в journal).

## Критерии неуспеха

Ни на одном из доступных символов OOS-метрики не проходят пороги после издержек
→ стратегия отклонена, пробуем следующую гипотезу (momentum US-сессии).
