# 02 — Data layer: исторические свечи в parquet

## Цель

Скачивать и хранить исторические OHLCV-свечи (1m/15m/1h) для бэктеста в parquet, с пагинацией и дедупликацией, через уже готовый адаптер (plan 01).

## Гипотеза edge'а

Edge'а нет — инфраструктура. Гипотеза проектная: REST-пагинация fetch_ohlcv даёт полную историю без дыр при правильном продвижении курсора since.

## Что входит

- `marketdata/candles.py`:
  - `timeframe_to_ms(tf)` — парс таймфрейма (1m/5m/15m/1h/4h/1d) в миллисекунды.
  - `async download_ohlcv(adapter, symbol, timeframe, start_ms, end_ms=None, page_limit=1000)` —
    пагинация вперёд по since, дедуп по timestamp, сортировка, стоп при пустой
    странице / отсутствии прогресса / достижении end_ms. Возвращает list[OHLCV].
  - `save_parquet(candles, path)` / `load_parquet(path) -> list[OHLCV]` — через pyarrow.
  - `candles_path(base_dir, exchange, symbol, timeframe) -> Path`.
- `scripts/download_klines.py` — CLI: `--exchange --symbol --timeframe --months` → скачать и сохранить.
- `gold-bot/.gitignore` — игнор `data/` (хранилище свечей не коммитится).

Хранение: `gold-bot/data/candles/{exchange}/{symbol}/{timeframe}.parquet` (symbol слеши → `_`).

## Данные / источники

- adapter.fetch_ohlcv (plan 01).
- pyarrow для parquet (изолирован на границе; наш код работает с list[OHLCV]).

## Метод проверки

- Unit: пагинация на моке адаптера (несколько страниц → пустая); дедуп перекрывающихся
  страниц; остановка по end_ms; timeframe_to_ms; parquet round-trip в tmp_path.
- Без сети (адаптер замокан). Реальная загрузка — ручная через скрипт.

## 10 причин почему может не сработать

1. Биржа отдаёт меньше свечей чем page_limit — пагинация должна останавливаться по «нет прогресса», не по размеру.
2. Дубли на границах страниц (since включительный) — дедуп по timestamp обязателен.
3. Бесконечный цикл если last_ts не растёт — явный брейк при отсутствии продвижения.
4. Неполная последняя свеча (текущий незакрытый интервал) — бэктест должен её игнорировать (отметим в плане 04).
5. Разные биржи — разные лимиты page_limit. Держим параметром.
6. Float в parquet — храним цены как string, чтобы не терять точность Decimal.
7. Часовые пояса/ms vs s — все timestamp в ms epoch (как ccxt).
8. Большой объём (1m за месяцы) — пишем стримом/батчами; для MVP держим в памяти (приемлемо до ~лет 1m).
9. Гэпы в истории (биржа лежала) — не заполняем искусственно; бэктест работает по факту.
10. pyarrow mypy-шум — изолируем override'ом, граница типизирована через list[OHLCV].

## Критерии успеха

- download_ohlcv правильно пагинирует и дедуплицирует (тесты).
- parquet round-trip сохраняет Decimal-точность (тест).
- ruff + format + mypy --strict чисто, pytest зелёный.
- скрипт download_klines запускается (ручной с сетью).
