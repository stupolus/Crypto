# Шаг 0 — разведка инструментов: лог решений

Дата: 2026-05-20. Автономный режим (goal). Фиксирую решения и ограничения.

## Что выяснилось

1. **Из dev-контейнера сеть до BingX и Bybit заблокирована** сетевой политикой
   окружения. Прямой публичный запрос `load_markets` к обеим биржам падает с
   `ExchangeNotAvailable` (open-api.bingx.com и api.bybit.com недоступны).
   → Живую разведку Шага 0 здесь выполнить нельзя.

2. **Решение:** Шаг 0 оформлен как runnable-артефакт — `scripts/recon_universe.py`.
   Запускается на VPS (где есть сеть), строит таблицу: класс/symbol/type/active/
   linear/taker/maker/lev_max/amt_min/price_prec по золоту, серебру и equity.
   Команда: `cd gold-bot && python -m scripts.recon_universe --md journal/step0-universe.md`.

## Регуляторные оговорки (резидент Испании/ЕЭЗ)

Публичный листинг ≠ доступность аккаунту. По состоянию знаний (требует подтверждения
авторизованным запросом / в UI):
- **Bybit:** TradFi/equity-перпы и часть продуктов в ЕЭЗ ограничены под MiCA/локальные
  правила. По equity на акции техногигантов в ЕЭЗ — высокий риск недоступности.
- **BingX:** перечень RWA/equity-перпов узкий и переменный; доступность по ЕЭЗ
  проверяется отдельно.
- **Золото/серебро:** наличие XAU/XAG/PAXG/XAUT-перпов и их доступность ЕЭЗ —
  только по факту recon, не по памяти. Тикеры не выдумываем (правило goal).

## Фактический юниверс: статус UNCONFIRMED

До запуска recon на VPS торгуемый юниверс **не зафиксирован**. План Б (если equity
недоступны в ЕЭЗ): фокус на токенизированном золоте/серебре, крипто-мейджоры (BTC/ETH/SOL)
как калибровочный набор инфраструктуры. Соответствует master-плану (10 причин, п.1-3,10).

## Что нужно от пользователя (для разблокировки live-ветки)

1. Запустить `recon_universe` на VPS → прислать/закоммитить `journal/step0-universe.md`.
2. Подтвердить доступность инструментов в UI биржи под своим KYC.
3. Ключи на VPS: только trading + read, **без вывода средств**, IP whitelist.
