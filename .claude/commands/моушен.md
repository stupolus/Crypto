# Моушен — анимация сайта на Framer Motion

Помоги добавить или поправить анимацию в дашборде `web/` (Vite + React 19) с помощью библиотеки **Framer Motion** (`framer-motion`, уже установлена в `web/package.json`).

Запрос пользователя: $ARGUMENTS

## Что делать

1. **Сначала пойми сцену.** Найди нужный компонент в `web/src/` (страницы — `web/src/pages/`, общие — `web/src/components/`, каркас — `web/src/Layout.tsx`). Прочитай его, прежде чем менять.
2. **Подключи `motion`.** Импорт: `import { motion, AnimatePresence } from "framer-motion"`. Заменяй обычные теги на `motion.*` (например `<div>` → `<motion.div>`) только там, где нужна анимация.
3. **Базовые паттерны:**
   - Появление: `initial={{ opacity: 0, y: 8 }} animate={{ opacity: 1, y: 0 }} transition={{ duration: 0.25 }}`.
   - Размонтирование: оборачивай в `<AnimatePresence>` и добавляй `exit={{ ... }}`.
   - Список: `staggerChildren` через `variants` на контейнере.
   - Ховер/тап: `whileHover`, `whileTap` (хорошо для кнопок и карточек).
   - Цифры/графики (`lightweight-charts`) не анимируй через motion — дёргается перерисовка.
4. **Уважай вкусы проекта.** Анимации сдержанные: длительность 0.15–0.35s, без резких bounce на финансовых данных. Дашборд про деньги, а не про спецэффекты.
5. **Проверь сборку:** `cd web && npm run build` (или хотя бы `tsc -b`) — не оставляй красный TypeScript.
6. **Доступность:** уважай `prefers-reduced-motion` — для крупных движений используй `useReducedMotion()` и отключай смещения.

## Подсказки по библиотеке

- React 19 совместим с `framer-motion@12`.
- Если анимация не запускается — проверь, что элемент именно `motion.*`, а не обычный тег.
- Для анимации между маршрутами (`react-router-dom`) оборачивай `<Routes>` в `<AnimatePresence mode="wait">` с `key={location.pathname}`.

В конце — короткое резюме на русском (было / стало) по правилам проекта.
