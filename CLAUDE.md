# CLAUDE.md

Главные правила. Читается каждой сессией — держим коротким. Детали живут в `AGENTS.md`, `бизнес/`, `plans/`.

## Проект

Автоматизированная торговля: крипта, RWA-перпы (золото, индексы), токенизированные акции. Цель — системно прибыльный бот 24/7 по проверенным правилам. Смелый, но считающий риск, а не «казино с плечом».

Принципы:
1. **Чужие сигналы — не триггеры.** Любой источник: парсим → статистика → проверка edge → встраиваем как фильтр/фичу. Не как вход.
2. **Размер позиции — от риска (% эквити), не от плеча.**
3. **Композит важнее сырого.** Один скринер = шум. Pump+OI+Volume+Funding+Heatmap = сигнал.
4. **Сначала план, потом код.** Любая фича — файл в `plans/`. Кода до плана нет.
5. **Каждое решение — в файлы.** Ничего «в голове Claude».

## Стек

Python 3.12+ (бот/парсеры/бэктест), TS-дашборд позже. Биржа №1 BingX, №2 Bybit (позже). RWA-данные Hyperliquid, heatmap Coinglass, транскрипция Whisper. Хранилище Parquet + SQLite/DuckDB → PostgreSQL по мере роста. Деплой VPS в Азии.

## Структура

- `бизнес/` — второй мозг (цели, риск-профиль, материалы). **Сначала `бизнес/INDEX.md`.**
- `plans/` — техпланы, один файл = одна фича.
- `retro/` — ретроспективы. `журнал/` — сделки и review.
- `parsers/` источники · `adapters/` биржи · `core/` инфра (risk/signal/backtest) · `strategies/` стратегии · `data/` данные · `ops/` мониторинг.

## Безопасность (жёстко)

- Реальная торговля — только по команде. Testnet после плана; live только после testnet + явного «да».
- Все ключи — через env, `.env*` в `.gitignore`, не коммитить. Ключи биржи: только торговля, без вывода, IP whitelist.
- Перед новой стратегией — критика «10 причин провала». Перед деплоем — security review.
- Не править `.claude/settings.json`, `CLAUDE.md`, `AGENTS.md` без явного «да» (сначала diff + обоснование).

## Git

Claude ведёт git сам в конце каждой осмысленной задачи: `commit` (понятное сообщение на русском) + `push` в feature-ветку (никогда не в `main`), PR на закрытой фазе. Авто-мерж только своих draft-PR при всех зелёных гейтах. **Полная механика, гейты и запреты — в `AGENTS.md` → «Git-процесс».**

## Трейдинг

Все цифры — **`бизнес/риск-профиль.md`** (единственный источник истины; код и планы ссылаются, не дублируют). Коротко: размер от риска в % эквити, изолированная маржа, стоп ставится вместе с входом, эффективное плечо ограничено, циркуит-брейкеры (день/неделя/месяц) автоматические. Скальпинг разрешён как отдельный профиль с жёсткими рамками — см. `бизнес/правила-торговли/правила-скальпинга.md`. Запрещено навсегда: 25x-казино, усреднение в убыток, кросс-маржа, позиция без биржевого стопа.

## Поток работы

1. Фича/стратегия → `plans/<NN>-<имя>-<дата>.md`.
2. План: первичный → исследование → адекватизация → 10 причин провала → фазы.
3. Реализация по фазам, каждая фаза = коммит. Любая стратегия — с бэктестом, иначе не реализована.
4. После задачи — резюме на русском (было / стало / какую проблему решал).
5. После важной сессии — `retro/<дата>-<тема>.md`.

## Памятка

- Контекст ≤60% — не перегружай, иначе новый чат. Один чат — одна задача.
- Код — только через агента, не руками.
- Длинные методологии — в `бизнес/материалы/`, здесь не дублировать.
