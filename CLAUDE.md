# CLAUDE.md

## Проект

Автоматизированная торговая платформа на крипту, RWA-перпы (золото, индексы) и токенизированные акции. Реалистичная цель: системно прибыльный бот, работающий 24/7 по проверенным правилам, а не «казино с плечом».

Принципы:
1. **Никаких чужих сигналов как триггеров.** Любой источник (TG-скринеры, новости, чужие каналы) проходит путь: парсим → накапливаем статистику → проверяем edge → встраиваем как фильтр или фичу. Не как триггер.
2. **Размер позиции от риска, не от плеча.** Риск считаем в % эквити, не в долларах.
3. **Композитный сигнал важнее сырого.** Один скринер = шум. Pump+OI+Volume+Funding+Heatmap = сигнал.
4. **Сначала план, потом код.** Любая фича — файл в `plans/`. Никакого кода до плана.
5. **Каждое решение — в файлы.** Ничего «в голове Claude» не остаётся.

## Стек (целевой)

- **Язык:** Python 3.12+ (бот, парсеры, бэктест), TypeScript (дашборд — позже).
- **Биржа №1:** BingX (там есть весь спектр — крипта, золото, индексы, акции).
- **Биржа №2 (позже):** Bybit для инструментов, которых нет на BingX, или для распределения капитала.
- **Источник данных по RWA-перпам:** Hyperliquid (~52% рынка RWA).
- **Heatmap ликвидаций:** Coinglass API.
- **Транскрипция видео-материалов:** Whisper.
- **Хранилище данных:** Parquet файлы + SQLite/DuckDB (на старте). PostgreSQL — когда понадобится.
- **Деплой:** VPS в Азии (близко к биржам).

## Правила безопасности (жёстко)

- **Никогда не пушь в `main`** без явной просьбы.
- **Никогда `--force`, `git reset --hard`, `--no-verify`** без явной просьбы.
- **Никогда не правь `.claude/settings.json`** без явной просьбы. Сначала diff и обоснование.
- **Никогда не запускай реальную торговлю без команды.** На testnet — после плана. На live — только после testnet + явного подтверждения.
- **Все API-ключи — через переменные окружения.** `.env*` в `.gitignore`. Никогда не коммитить ключи.
- API-ключи биржи: **только торговые права, без вывода средств.** IP whitelist обязателен.
- Перед запуском новой стратегии — критика «10 причин провала».
- Перед деплоем — security review.

## Git-процесс (зона ответственности Claude)

Пока пользователь нарабатывает опыт, всю механическую git-работу делает Claude автоматически в конце каждой осмысленной задачи:
- `git add <конкретные файлы>` (никогда `-A`, никогда `.`).
- `git commit` с понятным сообщением на русском.
- `git push` в feature-ветку (НЕ в `main`).
- Подготовка Pull Request через `gh pr create` с титулом и описанием на русском, когда фаза/задача закрыта целиком.

**Авто-мерж своих PR в `main`** (разрешено с 2026-05-11). Claude мержит draft PR, который сам же создал в текущей сессии, **только если все условия выполнены**:
1. Все unit-тесты зелёные (`pytest -q`).
2. Integration зелёные на VST (или skip с понятной причиной, зафиксированной в коммите/PR).
3. ruff + ruff format + mypy strict — чисто.
4. PR создан Claude в текущей сессии (не чужой).
5. PR не трогает `.claude/settings.json`, `CLAUDE.md`, `AGENTS.md`, `.env*`, секреты — для них всегда требуется явное согласие пользователя.
6. Не прямой push в `main`, всегда через `mcp__github__merge_pull_request`.

После авто-мержа — `git checkout main && git pull origin main`.

Жёсткие запреты остаются (никогда без явной команды пользователя):
- ❌ Прямой `git push origin main`.
- ❌ Мерж чужих PR.
- ❌ Мерж если хоть один из гейтов выше красный.
- ❌ `--force`, `git reset --hard`, `--no-verify`, `git rebase -i`, `git commit --amend`.
- ❌ Коммитить файлы с возможными секретами (`.env*`, `credentials*`, `*.key`, `*.pem`).

Ритм: каждая закрытая задача = `commit + push`. Каждая закрытая фаза/тема = PR с описанием для merge.

## Правила трейдинга (см. бизнес/риск-профиль.md для всех цифр)

- **Риск на сделку:** 1% эквити (для A+ сетапов до 2%).
- **Дневной стоп-лосс:** −3% эквити → бот выключается до завтра.
- **Недельный лимит:** −7% → пауза на неделю + ретроспектива.
- **Месячный лимит:** −15% → стоп проекта, разбор полётов.
- **Макс. плечо:** 5x эффективное (для A+ до 5x, обычно 1–3x).
- **Макс. одновременных позиций:** 1 на инструмент. По разным инструментам — пересчёт через корреляции.
- **Скальпинг:** запрет снят по явной инструкции владельца 2026-05-15.
  ⚠️ Снято ПРОТИВ задокументированных данных: скальпинг проверялся
  (издержки доминируют на 0.1% движениях, стакан невалидируем) —
  net-отрицателен. См. `бизнес/правила-торговли/анти-скальпинг.md`,
  retro/2026-05-15-поиск-edge-итог.md. Снятие правила НЕ авторизует
  live-скальпинг: остаются жёсткие гейты (бэктест + 4 нед demo +
  явная команда на реальную торговлю).

## Поток работы

1. Новая фича/стратегия → файл в `plans/<имя>-<дата>.md`.
2. План: первичный → исследование → адекватизация → 10 причин провала → разбиение на фазы.
3. Реализация фаза за фазой, каждая фаза заканчивается коммитом.
4. После завершённой задачи — резюме на русском: что было / что стало, какую проблему решал.
5. После важной сессии — `retro/<дата>-<тема>.md`.
