# AGENTS.md

## Подход

Этот проект — алгоритмическая торговля. Цена ошибки — реальные деньги. Поэтому работа агентов максимально дисциплинированная:

1. **Сначала понять, потом писать.** Перед любым кодом — план в `plans/`.
2. **Никаких допущений «как обычно».** Каждая биржа, каждый API имеет квирки — читать документацию, не полагаться на память.
3. **Любой числовой параметр (порог, лимит, плечо)** должен быть в конфиге, не в коде. Чтобы можно было крутить без риска что-то сломать.
4. **Любая стратегия — обязательно с бэктестом.** Без бэктеста стратегия не считается реализованной.
5. **Никогда не торговать на live без 4+ недель demo.**

## Принципы кода

- **Малые модули, ясные интерфейсы.** Adapter, Strategy, RiskEngine — каждый делает одну вещь.
- **Type hints везде** (Python 3.12, mypy strict).
- **Логи структурные** (json), уровни: DEBUG (для отладки), INFO (события), WARNING (аномалии), ERROR (ошибки), CRITICAL (стоп бота).
- **Тесты** на критичные места: размер позиции, риск-проверки, расчёт ликвидации, исполнение ордеров.
- **Никаких magic numbers.** Все пороги — в `core/config.yaml` со ссылкой на источник в `бизнес/`.

## Источник истины для правил

Все правила трейдинга, риск-параметры, инструменты, источники — в `бизнес/`. Если возник вопрос «какое плечо использовать», ответ ищется в `бизнес/риск-профиль.md`, а не выдумывается.

## Правила работы со знаниями

- При вопросе про риск, депозит, цели, инструменты — сначала `бизнес/INDEX.md`, оттуда нужный файл.
- **Никогда не хардкодить даты, цены, цифры** в коде или в файлах `бизнес/` (кроме `риск-профиль.md` и `цели.md`, где они и должны быть).
- Если нужная цифра отсутствует в риск-профиле — добавить туда, а не вставить число прямо в код.

## Анти-паттерны (не делать)

- **Не подгонять параметры под бэктест** (overfitting). Каждый параметр обоснован логикой, а не цифрой Sharpe.
- **Не добавлять фичи на ходу** во время живой торговли. Только: остановили → план → бэктест → деплой.
- **Не доверять чужим сигналам без статистики.** TG-скринеры и каналы — источник данных, не источник входов.
- **Не использовать high-frequency паттерны** в стратегиях, рассчитанных на интрадей. Латенси убьёт.
- **Не отлаживать на live.** Никогда.
